## Cross-Phase Integration Issues

1. **String interpolation stub** — Lexer, parser, resolver, and type checker all handle `IRON_NODE_INTERP_STRING` correctly, but `gen_exprs.c:291` emits `""` with a "Phase 3 will use snprintf" comment that was never fulfilled.

2. **Parallel-for parameter mismatch** — The chunk function is generated with `(int64_t start, int64_t end, void* ctx_arg)` signature, but `Iron_pool_submit` expects `void (*fn)(void*)`. The cast at `gen_stmts.c:292` makes the function callable but `start` and `end` receive uninitialized values.

3. **Stdlib unreachable** — All four stdlib modules (math, io, time, log) are compiled into every binary but no `import math` / `import io` / `import time` / `import log` mechanism exists to make their functions callable from Iron source. They need the same treatment as `import raylib` — wrapper .iron files with extern declarations.
