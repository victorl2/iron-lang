**Extend src/codegen/gen_exprs.c** — add lambda emission in the `IRON_NODE_LAMBDA` case of `emit_expr`:

Lambda codegen strategy (per CONTEXT.md naming: Iron_<parent>_lambda_N):
1. Generate unique name: `Iron_<enclosing_func>_lambda_<counter>` using `ctx->lambda_counter++`.
2. Analyze captures: walk the lambda body, find all idents that reference symbols from outer scopes.
3. If no captures (pure lambda):
   - Emit the lambda as a regular C function in `ctx->lifted_funcs`.
   - At the use site, emit the function pointer: `Iron_main_lambda_0`.
4. If has captures (closure):
   - Generate a closure env struct: `typedef struct { <captured_types> } Iron_main_lambda_0_env;`
   - Emit the lambda function with env pointer as first arg: `ret Iron_main_lambda_0(Iron_main_lambda_0_env* env, <params>)`
   - At the use site, allocate and populate env, then construct a closure struct:
     ```c
     Iron_main_lambda_0_env* env = malloc(sizeof(Iron_main_lambda_0_env));
     env->score = &score;
     ```
   - For non-escaping lambdas: stack-allocate env. For escaping (stored in object field): heap-allocate.

**Extend src/codegen/gen_stmts.c** — add spawn and parallel-for emission:

`IRON_NODE_SPAWN`:
1. Lift the spawn body to a named C function: `void Iron_spawn_<name>_<N>(void* arg)`
2. If the body captures variables, generate a context struct and pass as `arg`.
3. At the use site, emit:
   ```c
   // pool_submit is a runtime function (stub for Phase 2)
   Iron_spawn_autosave_0_ctx* ctx = malloc(sizeof(...));
   ctx->state = &state;
   Iron_pool_submit(pool, Iron_spawn_autosave_0, ctx);
   ```
4. If a handle is assigned: `Iron_Handle Iron_handle_name = Iron_pool_submit_handle(...);`
5. For `IRON_NODE_AWAIT`: `Iron_handle_wait(&handle);` — blocks until done.

`IRON_NODE_FOR` with `is_parallel`:
1. Lift the for-body to a chunk function: `void Iron_parallel_chunk_N(int64_t start, int64_t end, void* ctx_arg)`
2. Generate a context struct for captured variables (the array being iterated, etc.).
3. At the use site, emit range splitting and submission:
   ```c
   int64_t total = <range_end>;
   int64_t chunk_size = (total + num_threads - 1) / num_threads;
   for (int64_t c = 0; c < total; c += chunk_size) {
       Iron_parallel_chunk_N_ctx* chunk_ctx = ...;
       chunk_ctx->start = c;
       chunk_ctx->end = (c + chunk_size > total) ? total : c + chunk_size;
       Iron_pool_submit(pool, Iron_parallel_chunk_N, chunk_ctx);
   }
   Iron_pool_barrier(pool);
   ```

Note: `Iron_pool_submit`, `Iron_pool_barrier`, `Iron_handle_wait`, `Iron_Handle` are runtime types/functions that will be implemented in Phase 3. For Phase 2, emit the correct C code referencing them; the generated C will compile when linked against the runtime.
