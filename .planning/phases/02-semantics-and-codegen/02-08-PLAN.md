**Bug 1 — IRON_NODE_FIELD_ACCESS (gen_exprs.c lines 433-437):**

Replace the unconditional `.` emission with pointer-aware logic. The object expression's `resolved_type` determines the accessor:

```c
case IRON_NODE_FIELD_ACCESS: {
    Iron_FieldAccess *fa = (Iron_FieldAccess *)node;
    emit_expr(sb, fa->object, ctx);

    /* Determine whether to use -> or . based on receiver type.
     * Methods receive self as T* (pointer), so field access is ->
     * For struct values (stack objects), use .
     * Heuristic: if the object is "self" identifier, always use -> because
     * methods declare self as T* (per emit_method_impl signature). */
    bool use_arrow = false;
    if (fa->object->kind == IRON_NODE_IDENT) {
        Iron_Ident *id = (Iron_Ident *)fa->object;
        if (strcmp(id->name, "self") == 0) {
            use_arrow = true;
        }
    }
    /* Also check resolved_type: if it's a pointer/heap type, use -> */
    /* For Phase 2, self is the only pointer receiver; stack objects use . */

    iron_strbuf_appendf(sb, "%s%s", use_arrow ? "->" : ".", fa->field);
    break;
}
```

**Bug 2 — IRON_NODE_CALL type-constructor redirect (gen_exprs.c lines 354-396):**

Inside the `IRON_NODE_CALL` handler, after the print/println special case and before the regular function call path, add a check: if the callee is an IRON_NODE_IDENT and its `resolved_sym->sym_kind == IRON_SYM_TYPE`, then this is a type-constructor call that should emit a compound literal (same as IRON_NODE_CONSTRUCT) instead of a C function call.

```c
/* After println check, before regular function call: */
/* Check if callee is a type name (object constructor) */
if (callee_id->resolved_sym &&
    callee_id->resolved_sym->sym_kind == IRON_SYM_TYPE) {
    /* Emit as compound literal, same as IRON_NODE_CONSTRUCT */
    const char *mangled = iron_mangle_name(callee_id->name, ctx->arena);
    iron_strbuf_appendf(sb, "(%s){", mangled);
    /* Look up field names from the object declaration */
    Iron_Symbol *sym = iron_scope_lookup(ctx->global_scope, callee_id->name);
    if (sym && sym->decl_node &&
        sym->decl_node->kind == IRON_NODE_OBJECT_DECL) {
        Iron_ObjectDecl *od = (Iron_ObjectDecl *)sym->decl_node;
        int n = call->arg_count < od->field_count ? call->arg_count : od->field_count;
        for (int i = 0; i < n; i++) {
            if (i > 0) iron_strbuf_appendf(sb, ", ");
            Iron_Field *f = (Iron_Field *)od->fields[i];
            iron_strbuf_appendf(sb, ".%s = ", f->name);
            emit_expr(sb, call->args[i], ctx);
        }
    } else {
        /* Fallback: positional init */
        for (int i = 0; i < call->arg_count; i++) {
            if (i > 0) iron_strbuf_appendf(sb, ", ");
            emit_expr(sb, call->args[i], ctx);
        }
    }
    iron_strbuf_appendf(sb, "}");
    break;
}
```

**Gap 2 — auto_free consumption (gen_stmts.c emit_block):**

After `emit_defers(sb, ctx, ctx->defer_depth - 1);` and before `ctx->indent--;`, add a scan over the block's statements looking for IRON_NODE_VAL_DECL and IRON_NODE_VAR_DECL whose `init` is an IRON_NODE_HEAP with `auto_free=true`. For each match, emit `free(varname);`.

```c
/* Emit free() for auto_free heap allocations in this block */
for (int i = 0; i < block->stmt_count; i++) {
    Iron_Node *s = block->stmts[i];
    const char *var_name = NULL;
    Iron_Node *init_expr = NULL;

    if (s->kind == IRON_NODE_VAL_DECL) {
        Iron_ValDecl *vd = (Iron_ValDecl *)s;
        var_name = vd->name;
        init_expr = vd->init;
    } else if (s->kind == IRON_NODE_VAR_DECL) {
        Iron_VarDecl *vd = (Iron_VarDecl *)s;
        var_name = vd->name;
        init_expr = vd->init;
    }

    if (init_expr && init_expr->kind == IRON_NODE_HEAP) {
        Iron_HeapExpr *he = (Iron_HeapExpr *)init_expr;
        if (he->auto_free) {
            codegen_indent(sb, ctx->indent);
            iron_strbuf_appendf(sb, "free(%s);\n", var_name);
        }
    }
}
```

1. Calls `run_codegen()` with a comprehensive Iron program that exercises all three fixed bugs:

   }

   }
   func main() {

2. Writes the generated C string to a temporary file (`/tmp/iron_test_XXXXXX.c` via mkstemp or a fixed path like `/tmp/iron_codegen_test.c`).
3. Runs `clang -std=c11 -Wall -Werror -fsyntax-only /tmp/iron_codegen_test.c` via `system()` or `popen()`.
4. Asserts the clang command returns exit code 0.
5. Cleans up the temp file.

This is the definitive regression test: if any of the three bugs recur, clang will reject the output and this test fails.

Note: the generated C includes `#include <stdint.h>`, `#include <stdbool.h>`, `#include <stdio.h>`, and `#include <stdlib.h>` so clang will find all needed headers. The `-fsyntax-only` flag means no linker is invoked (we don't need a main() that actually runs since prototypes may reference runtime stubs like `Iron_pool_submit` that don't exist yet).

However, `-fsyntax-only` still type-checks function bodies and calls. Functions like `Iron_pool_submit` and `Iron_pool_barrier` are referenced but not declared — this will cause errors. To handle this, the clang invocation should use `-Wno-implicit-function-declaration` (or alternatively, the test program should avoid constructs that emit pool/spawn stubs). The simplest approach: use a test program that only exercises the three fixed bugs (field access, construction, println) without spawn/parallel. The program above (Vec2 + method + println) is sufficient.

Actually, check whether `-fsyntax-only` with `-Werror` will error on undeclared functions like printf's implicit declaration. Since the codegen emits `#include <stdio.h>`, printf is declared. The test program above uses only: structs, functions, printf — all of which will be self-contained in the generated C. This should work.

Register the test in main().
