**src/analyzer/escape.h:**
```c
#ifndef IRON_ESCAPE_H
#define IRON_ESCAPE_H

#include "parser/ast.h"
#include "analyzer/scope.h"
#include "analyzer/types.h"
#include "diagnostics/diagnostics.h"
#include "util/arena.h"

/* Run escape analysis on a type-checked program.
 * Sets auto_free and escapes flags on Iron_HeapExpr nodes.
 * Emits E0207 for escaping heap values without free/leak.
 * Emits E0212/E0213/E0214 for invalid free/leak usage.
 */
void iron_escape_analyze(Iron_Program *program, Iron_Scope *global_scope,
                         Iron_Arena *arena, Iron_DiagList *diags);

#endif
```

**src/analyzer/escape.c:**
Create `EscapeCtx` struct:
- `Iron_Arena *arena`, `Iron_DiagList *diags`, `Iron_Scope *current_scope`
- `Iron_Node **heap_values` — stb_ds dynamic array of heap allocations in current function
- `const char **freed_names` — stb_ds array of names that have been freed
- `const char **leaked_names` — stb_ds array of names that have been leaked
- `int function_depth` — to track function scope boundaries

Algorithm (intra-procedural + one-level callee per CONTEXT.md):

1. Walk each function/method body:
   a. Collect all `IRON_NODE_HEAP` expressions and their binding names (from enclosing ValDecl/VarDecl).
   b. Collect all `IRON_NODE_FREE` targets and `IRON_NODE_LEAK` targets.
   c. For each heap allocation, determine if it escapes:
      - Escapes if: the bound variable appears in a `IRON_NODE_RETURN` value expression
      - Escapes if: the bound variable is assigned to a field of an object from an outer scope (e.g., `game.enemies.add(e)`)
      - Escapes if: the bound variable is passed to a function parameter that stores it (one-level callee tracking: if the callee's param is `var` and the callee stores it in an outer scope)
      - For simplicity in Phase 2: if the heap variable is passed to any function call and that function is not a known pure function, mark as "potentially escaping" and check if free/leak is present.
   d. For each heap allocation that does NOT escape: set `heap_node->auto_free = true`.
   e. For each heap allocation that DOES escape: check if it has a corresponding `free` or `leak`. If neither: emit E0207. Set `heap_node->escapes = true`.

2. Validate `free` statements:
   - Resolve the target. If the target was not allocated with `heap`, emit E0212.

3. Validate `leak` statements:
   - If target was not allocated with `heap`, emit E0213.
   - If target type is IRON_TYPE_RC, emit E0214.

4. Skip `rc` values entirely — they manage their own lifetime.

**src/analyzer/concurrency.h:**
```c
#ifndef IRON_CONCURRENCY_H
#define IRON_CONCURRENCY_H

#include "parser/ast.h"
#include "analyzer/scope.h"
#include "analyzer/types.h"
#include "diagnostics/diagnostics.h"
#include "util/arena.h"

/* Run concurrency checks on a type-checked program.
 * Validates parallel-for bodies do not mutate outer non-mutex variables.
 * Validates spawn block captures (future: more checks).
 */
void iron_concurrency_check(Iron_Program *program, Iron_Scope *global_scope,
                            Iron_Arena *arena, Iron_DiagList *diags);

#endif
```

**src/analyzer/concurrency.c:**
Create `ConcurrencyCtx`:
- `Iron_Arena *arena`, `Iron_DiagList *diags`
- `Iron_Scope *parallel_scope` — the scope of the parallel-for body (set when entering a parallel block)
- `bool in_parallel` — flag for whether we're inside a parallel-for body

Algorithm:
1. Walk the AST. When encountering a `IRON_NODE_FOR` with `is_parallel=true`:
   a. Set `in_parallel = true`, record the for-body scope.
   b. Walk the body looking for assignment statements (`IRON_NODE_ASSIGN`, compound assignments `+=` etc.).
   c. For each assignment target, resolve it to its symbol. Check if the symbol was defined in a scope OUTSIDE the parallel-for body scope (i.e., scope chain walk from the for-body's scope to the symbol's defining scope crosses the parallel boundary).
   d. If the outer variable's type is not a mutex type (IRON_TYPE_OBJECT with name matching "mutex" pattern or wrapped in mutex), emit E0208 with message "cannot mutate outer variable '{name}' in parallel for body; use mutex for shared state".
   e. Reading outer variables is fine (only mutations are checked).

2. For `IRON_NODE_SPAWN` blocks (lighter check for Phase 2):
   - No additional validation beyond what the type checker already provides. Full spawn capture validation can be extended in Phase 3.
