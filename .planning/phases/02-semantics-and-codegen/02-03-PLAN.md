<objective>
Implement the type checking pass that annotates every expression with its resolved type, enforces type safety, val immutability, nullable narrowing, and interface completeness.

Purpose: Type checking is the second semantic pass. It transforms a name-resolved AST into a fully type-annotated AST. Every expression node gets a `resolved_type` pointer. Type errors (mismatches, nullable access, val reassignment, missing interface methods) are reported with precise spans. The annotated AST is the input to escape analysis and codegen.

Output: src/analyzer/typecheck.h/c with iron_typecheck() entry point, Unity tests covering type inference, type errors, nullable narrowing, interface validation.

**src/analyzer/typecheck.c:**
Create a `TypeCheckCtx` struct:
- `Iron_Arena *arena`
- `Iron_DiagList *diags`
- `Iron_Scope *current_scope`
- `Iron_Type *current_return_type` (expected return type of current function; NULL outside functions)
- `const char *current_method_type` (owning type name if in method, NULL otherwise)
- Narrowing map: stb_ds hash map `{ char *key; Iron_Type *value; } *narrowed;` mapping symbol names to their narrowed types

Implement type checking as a recursive switch-dispatch walk over all AST node kinds.

**Type annotation resolution** (`resolve_type_annotation` helper):
Given an `Iron_TypeAnnotation` node, look up the name in scope:
- Primitive names ("Int", "Int8", ..., "Float", "Float32", "Float64", "Bool", "String"): return corresponding `iron_type_make_primitive()`
- If `is_nullable`: wrap result in `iron_type_make_nullable()`
- If `is_array`: wrap in `iron_type_make_array()` with size from `array_size`
- Object/enum/interface names: look up in scope, return associated type
- Generic args: recursively resolve and create monomorphized type name

**Expression type inference** (`check_expr` returning `Iron_Type*`):
- `IRON_NODE_INT_LIT`: return IRON_TYPE_INT
- `IRON_NODE_FLOAT_LIT`: return IRON_TYPE_FLOAT
- `IRON_NODE_BOOL_LIT`: return IRON_TYPE_BOOL
- `IRON_NODE_STRING_LIT` / `IRON_NODE_INTERP_STRING`: return IRON_TYPE_STRING
- `IRON_NODE_NULL_LIT`: return IRON_TYPE_NULL
- `IRON_NODE_IDENT`: return `ident->resolved_sym->type` (if resolved_sym is non-NULL). Check narrowing map for nullable narrowing.
- `IRON_NODE_BINARY`: check left and right. For arithmetic ops (+,-,*,/,%): both must be same numeric type, result is that type. For comparison ops (==,!=,<,>,<=,>=): both must be compatible, result is IRON_TYPE_BOOL. For `and`/`or`: both must be IRON_TYPE_BOOL, result is IRON_TYPE_BOOL. If operands don't match: E0202.
- `IRON_NODE_UNARY`: for `not`: operand must be BOOL, result is BOOL. For `-`: operand must be numeric, result is same type.
- `IRON_NODE_CALL`: check callee type is IRON_TYPE_FUNC. Validate arg count (E0216) and arg types (E0217). Return the function's return_type.
- `IRON_NODE_METHOD_CALL`: look up method on the object's type. Validate args. Return method return type.
- `IRON_NODE_FIELD_ACCESS`: resolve object type to an object decl, find the field by name. If not found: E0219. Return field type.
- `IRON_NODE_CONSTRUCT`: look up type_name. If SYM_TYPE: validate arg count matches field count, validate arg types match field types in order. Return the object type. If SYM_FUNCTION: rewrite as call (change kind to IRON_NODE_CALL or handle as function call). If neither: E0218.
- `IRON_NODE_HEAP`: check inner, return same type. Mark as heap-allocated.
- `IRON_NODE_RC`: check inner, return `iron_type_make_rc(inner_type)`.
- `IRON_NODE_LAMBDA`: push scope, define params, check body, infer return type. Return IRON_TYPE_FUNC.
- `IRON_NODE_IS`: check expr, return IRON_TYPE_BOOL.
- `IRON_NODE_INDEX`: check object is array/list type, return element type.
- `IRON_NODE_SLICE`: return same array type.
- `IRON_NODE_AWAIT`: return the handle's result type (stub: IRON_TYPE_VOID for now).

Set `node->resolved_type` on every expression node after computing its type.

**Statement checking** (`check_stmt`):
- `IRON_NODE_VAL_DECL`: if type_ann present, resolve it; check init expr type matches. If no type_ann, declared_type = init's resolved_type. Set `sym->type = declared_type`. Mark `sym->is_mutable = false`.
- `IRON_NODE_VAR_DECL`: same but `is_mutable = true`.
- `IRON_NODE_ASSIGN`: look up target symbol. If `!sym->is_mutable`, emit E0203 (val reassignment). Check value type matches target type (E0202).
- `IRON_NODE_RETURN`: check value type matches `current_return_type` (E0215).
- `IRON_NODE_IF`: check condition is IRON_TYPE_BOOL. **Nullable narrowing:** if condition is `x != null`, save narrowing state, narrow x to non-nullable in then-block. If condition is `x == null`, narrow x in else-block and in continuation if then-block always returns. **Is-narrowing:** if condition is `e is Player`, narrow e to Player in then-block. Restore narrowing state after block.
- `IRON_NODE_FOR`: push scope, define var_name, check iterable, check body.
- `IRON_NODE_WHILE`: check condition is BOOL, check body.
- `IRON_NODE_MATCH`: check subject, check each case pattern type-compatible.

**Interface completeness** (after all methods resolved):
For each `Iron_ObjectDecl` with `implements_count > 0`:
- For each interface name, look up the interface's method_sigs.
- For each method sig, check that a MethodDecl exists with matching name, param types, and return type for this object type.
- If any method is missing: emit E0205 with the missing method name and interface name.

**No implicit numeric conversions** (per CONTEXT.md locked decision):
If a binary expr has Int on one side and Float on the other, emit E0222 (IRON_ERR_NUMERIC_CONVERSION) with suggestion "Use explicit cast: Float(x)".
