<objective>
Implement two-pass name resolution that builds the scope tree and resolves all identifiers to their declarations.

Purpose: Name resolution is the first semantic pass. It builds the scope tree, populates symbol tables, and links every identifier reference to its declaration. Without this, type checking cannot determine what any name refers to. The two-pass approach (collect-then-resolve) handles Iron's forward reference pattern where methods can appear before their owning object declarations.

Output: src/analyzer/resolve.h/c with iron_resolve() entry point, ast.h extended with resolved_sym field, Unity tests proving resolution works.
</objective>

**src/analyzer/resolve.c:**
Implement the two-pass resolution algorithm from RESEARCH.md Pattern 2.

Create a `ResolveCtx` struct holding: `Iron_Arena *arena`, `Iron_DiagList *diags`, `Iron_Scope *global_scope`, `Iron_Scope *current_scope`, `Iron_MethodDecl *current_method` (NULL if not in a method), `const char *current_type_name` (NULL if not in a method).

**Pass 1a — Collect top-level declarations:**
Walk `program->decls[]` and for each:
- `IRON_NODE_OBJECT_DECL`: create SYM_TYPE symbol, define in global scope. Also create a type with `iron_type_make_object`.
- `IRON_NODE_INTERFACE_DECL`: create SYM_INTERFACE symbol, define in global scope.
- `IRON_NODE_ENUM_DECL`: create SYM_ENUM symbol, define in global scope. Also define each variant as SYM_ENUM_VARIANT.
- `IRON_NODE_FUNC_DECL`: create SYM_FUNCTION symbol, define in global scope.
- `IRON_NODE_IMPORT_DECL`: record import path. For Phase 2, just store imports in the context (actual file resolution is Phase 3 CLI). If alias is present, define alias as a symbol.

**Pass 1b — Attach methods to types:**
Walk `program->decls[]` again for `IRON_NODE_METHOD_DECL`:
- Look up `type_name` in global scope. If not found, emit IRON_ERR_UNDEFINED_VAR.
- If found and is SYM_TYPE, set `method->owner_sym` to the found symbol.
- Register method in a methods table (stb_ds hash map keyed by "TypeName.method_name").

**Pass 2 — Resolve identifiers:**
Walk the entire AST using a recursive dispatch (direct switch on node kind, similar to printer.c pattern — NOT iron_ast_walk, because we need scope push/pop control).

Key resolution rules:
- On entering `IRON_NODE_FUNC_DECL`: push FUNCTION scope, define params as SYM_PARAM.
- On entering `IRON_NODE_METHOD_DECL`: push FUNCTION scope, define implicit `self` as SYM_VARIABLE (type = owning object type, is_mutable = true), define params as SYM_PARAM. Set `current_method` and `current_type_name`.
- On entering `IRON_NODE_BLOCK`: push BLOCK scope.
- On `IRON_NODE_VAL_DECL`: resolve init expression, then define name as SYM_VARIABLE (is_mutable=false) in current scope.
- On `IRON_NODE_VAR_DECL`: resolve init expression, then define name as SYM_VARIABLE (is_mutable=true) in current scope.
- On `IRON_NODE_FOR`: push BLOCK scope, define var_name as SYM_VARIABLE, resolve iterable and body.
- On `IRON_NODE_IDENT`: look up name in current scope chain. If found, set `ident->resolved_sym = sym`. If not found, emit IRON_ERR_UNDEFINED_VAR with span.
- On `IRON_NODE_IDENT` with name == "self": if not inside a method, emit IRON_ERR_SELF_OUTSIDE_METHOD.
- On `IRON_NODE_IDENT` with name == "super": if inside a method, check if owning type has extends_name. If not, emit IRON_ERR_SUPER_NO_PARENT.
- On `IRON_NODE_CONSTRUCT`: look up type_name in scope chain. (Disambiguation of construct vs call happens in type checker, not here.)
- On exiting any scope: pop scope (just set current_scope = current_scope->parent).

Handle duplicate declarations: `iron_scope_define` returns false. On false, emit IRON_ERR_DUPLICATE_DECL.
