Locked decisions from CONTEXT.md:
- Timer field names: elapsed_ms (Int), duration_ms (Int)
- time.Timer(2.0): constructor takes Float seconds, converts to ms internally
- timer.update(dt): dt is Float seconds, adds (int64_t)(dt * 1000.0) to elapsed_ms
- timer.reset(): zeroes elapsed_ms, returns void (NOT a new Timer)
- timer.done(): returns elapsed_ms >= duration_ms as Bool
- time.since(start): takes Float (seconds from time.now()), returns Float elapsed seconds

Name mangling for new API:
  Time.Timer(duration)  -> Iron_time_Timer(double duration_s)   (constructor on Time type)
  Timer.done(t)         -> Iron_timer_done(Iron_Timer t)
  Timer.update(t, dt)   -> Iron_timer_update(Iron_Timer *t, double dt)  [pointer for mutation]
  Timer.reset(t)        -> Iron_timer_reset(Iron_Timer *t)               [pointer for mutation]
  Time.since(start)     -> Iron_time_since(double start)

PITFALL — Timer mutation (from RESEARCH.md Pitfall 1):

and have the call site assign back — but this conflicts with the void return type. Investigate
by reading src/codegen/hir_to_lir.c method dispatch section to understand pointer-passing
behavior for struct methods before finalizing the update/reset signatures.

Replace `src/stdlib/iron_time.c` entirely:

```c
#include "iron_time.h"
#include <stdlib.h>

/* ── Wall-clock time (seconds) ───────────────────────────────────────────── */

double Iron_time_now(void) {
    struct timespec ts;
    clock_gettime(CLOCK_REALTIME, &ts);
    return (double)ts.tv_sec + (double)ts.tv_nsec / 1.0e9;
}

/* ── Monotonic time (milliseconds) ──────────────────────────────────────── */

int64_t Iron_time_now_ms(void) {
    struct timespec ts;
    clock_gettime(CLOCK_MONOTONIC, &ts);
    return (int64_t)ts.tv_sec * 1000 + (int64_t)ts.tv_nsec / 1000000;
}

/* ── Sleep ───────────────────────────────────────────────────────────────── */

void Iron_time_sleep(int64_t ms) {
    struct timespec ts;
    ts.tv_sec  = (time_t)(ms / 1000);
    ts.tv_nsec = (long)((ms % 1000) * 1000000L);
    nanosleep(&ts, NULL);
}

/* ── Elapsed time ────────────────────────────────────────────────────────── */

double Iron_time_since(double start) {
    return Iron_time_now() - start;
}

/* ── Timer (accumulator style) ───────────────────────────────────────────── */

Iron_Timer Iron_time_Timer(double duration_s) {
    Iron_Timer t;
    t.elapsed_ms  = 0;
    t.duration_ms = (int64_t)(duration_s * 1000.0);
    return t;
}

bool Iron_timer_done(Iron_Timer t) {
    return t.elapsed_ms >= t.duration_ms;
}

void Iron_timer_update(Iron_Timer *t, double dt) {
    t->elapsed_ms += (int64_t)(dt * 1000.0);
}

void Iron_timer_reset(Iron_Timer *t) {
    t->elapsed_ms = 0;
}
```

Key points:
- `Iron_time_Timer` is the constructor called by `time.Timer(2.0)` (Type=Time, method=Timer).
- `Iron_timer_update` and `Iron_timer_reset` take `Iron_Timer *t` for in-place mutation.
- The IRON_TIMER_STRUCT_DEFINED guard is preserved with the updated field names (elapsed_ms, duration_ms).
- `<stdbool.h>` is added to iron_time.h for the `bool` return type of Iron_timer_done.
