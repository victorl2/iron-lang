<objective>
Add a dead alloca elimination pass to the LIR optimizer that removes phi-origin allocas with zero live loads, along with all their stores.

Purpose: After phi_eliminate + copy-propagation, some phi-origin allocas are written to but never read (their loads were already propagated away by copy-prop). DCE cannot remove them because ALLOCA is treated as non-pure and STORE is always kept as side-effecting. A targeted pass removes these dead allocas and their stores, reducing generated C temporaries.

Output: Modified `src/lir/lir_optimize.c` with `run_dead_alloca_elimination` function called post-fixpoint, producing measurably fewer temporaries in connected_components and simpler functions with phi nodes.
</objective>

The function signature: `static bool run_dead_alloca_elimination(IronLIR_Module *module)`

Algorithm (3 scans per function):

**Scan 1 — Mark "loaded" allocas:**
For each instruction in every block, mark an alloca ID as "loaded" (alive) if:
- A `LOAD` instruction references it as `in->load.ptr`
- A `GET_INDEX` instruction references it as `in->index.array`
- A `SET_INDEX` instruction references it as `in->index.array`
- A `GET_FIELD` instruction references it as `in->field.object`
- A `SET_FIELD` instruction references it as `in->field.object`

Use a stb_ds hashmap: `struct { IronLIR_ValueId key; bool value; } *loaded = NULL;` and `hmput(loaded, id, true)`.

**Scan 1b — Escape safety check:**
Call `compute_escape_set(fn)`. For any alloca that is in the escape set with `value == true`, also mark it as "loaded" (do not remove escaped allocas even if they have zero loads). This is a one-line loop:
```c
for (ptrdiff_t ei = 0; ei < hmlen(escape_set); ei++) {
    if (escape_set[ei].value) {
        hmput(loaded, escape_set[ei].key, true);
    }
}
hmfree(escape_set);
```

**Scan 2 — Identify dead allocas:**
For each ALLOCA instruction, check `hmgeti(loaded, in->id) < 0`. If not loaded, put into a `dead_alloca` hashmap.

**Scan 3 — Remove dead allocas and their stores:**
For each block, compact instructions: keep instruction unless:
- `in->kind == IRON_LIR_ALLOCA && hmgeti(dead_alloca, in->id) >= 0`
- `in->kind == IRON_LIR_STORE && hmgeti(dead_alloca, in->store.ptr) >= 0`

For removed instructions, null out `fn->value_table[in->id]` if `in->id != IRON_LIR_VALUE_INVALID && (ptrdiff_t)in->id < arrlen(fn->value_table)`. Set `changed = true`. Update `blk->instr_count = new_count`.

Free both hashmaps after processing each function.

**Pass placement in iron_lir_optimize:**
Insert a call to `run_dead_alloca_elimination(module)` AFTER the fixpoint loop exits (after the `iron_diaglist_free(&verify_diags)` call at line ~3302) and BEFORE `iron_lir_compute_inline_info` (line ~3306). Add a `dump_passes` block matching the existing pattern:

```c
}
```
