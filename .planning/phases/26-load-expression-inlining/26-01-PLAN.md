<objective>
Remove the blanket LOAD exclusion from expression inlining eligibility, enabling same-block LOADs to be inlined as sub-expressions at their use sites. Cross-block LOADs remain excluded by the existing cross-block guard.

Purpose: Eliminate one layer of unnecessary copy temporaries per function, reducing generated C verbosity and improving readability. This was a conservative safety guard added in v0.0.5 that is no longer needed because the cross-block exclusion logic already handles the dangerous case.

Output: Modified optimizer (one line removed), new regression test, full test suite green.
</objective>

    if (kind == IRON_LIR_MAKE_CLOSURE) return false;

}
