Note: `Image.to_texture(img: Image) -> Texture` is the Iron method for raylib's `LoadTextureFromImage(image)` — the image-to-texture bridge. Plan 66-02's Image.* lives in Image namespace; this method is added alongside (Iron name `Image.to_texture` preserves the receiver semantic, dispatches to Iron_image_to_texture).

`Texture.set_filter` and `Texture.set_wrap` take Texture by value; raylib's `SetTextureFilter`/`SetTextureWrap` take Texture by value too, so the Iron method does NOT return — the GPU state is mutated on the C side (Texture's `id` is what raylib uses as the handle; the struct is just a handle wrapper).

`Texture.gen_mipmaps` returns Texture because `GenTextureMipmaps(Texture *)` is the one truly mutating call that needs the return-by-value pattern.
