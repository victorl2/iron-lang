Make sure the cleanup happens on all exit paths (success and error). Follow the existing error-handling pattern in build.c.

**3. Update --verbose section (around line 727-730):**
Replace:
```c
if (opts.verbose) {
    fprintf(stdout, "=== Generated C ===\n%s\n=== End Generated C ===\n",
            c_src);
}
```

With (print IR first, then C -- per user decision):
```c
if (opts.verbose) {
    /* Print IR before phi elimination (clean SSA form) */
    /* Note: iron_ir_emit_c modifies IR in-place for phi elimination,
     * so we print IR BEFORE calling emit_c. Move this before emit_c call. */
    ...
}
```

IMPORTANT: The IR print must happen BEFORE `iron_ir_emit_c()` is called, because emit_c modifies the IR in-place for phi elimination. So restructure the code flow:
1. Lower AST to IR
2. If verbose: print IR
3. Call iron_ir_emit_c (modifies IR in-place)
4. If verbose: print generated C
5. Continue with write_temp_c / invoke_clang

The verbose IR output format:
```c
if (opts.verbose && ir_module) {
    char *ir_text = iron_ir_print(ir_module, true);
    if (ir_text) {
        fprintf(stdout, "=== Generated IR ===\n%s\n=== End IR ===\n\n", ir_text);
        free(ir_text);
    }
}
```

After emit_c:
```c
if (opts.verbose && c_src) {
    fprintf(stdout, "=== Generated C ===\n%s\n=== End Generated C ===\n", c_src);
}
```
