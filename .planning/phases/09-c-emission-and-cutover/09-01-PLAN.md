  <action>
Replace the stub in `src/ir/emit_c.c` with the full IR-to-C emission backend. The implementation must follow the section-based StrBuf output model from the old codegen. Key components:

**1. EmitCtx struct** with these sections (matching old codegen order):
- includes, forward_decls, struct_bodies, enum_defs, global_consts, prototypes, lifted_funcs, implementations, main_wrapper
- Additional state: arena, diags, module, indent, next_type_tag (starts at 1), emitted_optionals (stb_ds string array), mono_registry (stb_ds string map)

**2. Phi elimination pre-pass** (`phi_eliminate()`):
- Walk all functions and blocks. For each IRON_IR_PHI instruction:
  - Replace the phi with an IRON_IR_ALLOCA of the phi's type in the entry block (or current block)
  - In each predecessor block listed in phi.pred_blocks, insert an IRON_IR_STORE before the terminator: store phi.values[i] into the alloca's ValueId
  - Change phi instruction to a LOAD from the alloca
- After elimination, assert no IRON_IR_PHI instructions remain
- Modify IR in-place (IR is consumed and freed after emission per user decision)

**3. Include emission** (exact match to old codegen lines 259-274):
```c
#include "runtime/iron_runtime.h"
#include <stdint.h>
#include <stdbool.h>
#include <stdlib.h>
#include <string.h>
#include <stdio.h>
#include "stdlib/iron_math.h"
#include "stdlib/iron_io.h"
#define IRON_TIMER_STRUCT_DEFINED
#include "stdlib/iron_time.h"
#include "stdlib/iron_log.h"
```

**4. Type declaration emission** from `module->type_decls[]`:
- Forward declarations: `typedef struct Iron_X Iron_X;` for objects and interfaces
- Struct bodies with topological sort (port `topo_visit()` from codegen.c or implement DFS on field dependencies). Each struct body includes `int32_t iron_type_tag;` as first field (for objects with inheritance). Emit `#define IRON_TAG_Iron_X N` after each struct body using next_type_tag counter.
- Enum defs: `typedef enum { Iron_E_V0, Iron_E_V1 } Iron_E;`
- Interface vtable structs: `typedef struct Iron_I_vtable { ... } Iron_I_vtable;` and `typedef struct { void *object; Iron_I_vtable *vtable; } Iron_I_ref;`
- Optional structs: `ensure_optional_type()` called lazily when nullable types encountered
- Monomorphized generic structs: check mono_registry, emit stub struct if new

**5. Function prototype emission**:
- For each non-extern function in module->funcs[]: emit `<return_type> <fn_name>(<params>);`
- Use `iron_type_to_c()` for return and param types. Note: `iron_type_to_c()` takes an `Iron_Codegen*` second arg -- you need to either create a minimal compatibility shim or call the underlying type mapping directly. The simplest approach: create a local `emit_type_str()` helper that does the same type-to-C mapping without needing Iron_Codegen (read gen_types.c to understand the mapping: IRON_TYPE_INT->"int64_t", IRON_TYPE_FLOAT->"double", IRON_TYPE_BOOL->"bool", IRON_TYPE_STRING->"Iron_String", IRON_TYPE_VOID->"void", nullable->optional struct, rc->pointer, object->pointer, etc.)

**6. Function body emission** (`emit_func()`):
- Emit signature: `<ret> <name>(<params>) {`
- Emit blocks in order (blocks[0] is always entry). Each block: `<label>:;\n` then instructions.
- Instruction dispatch (`emit_instr()`) -- switch on IronIR_InstrKind for all 43+ cases:
  - **Constants**: `int64_t _v1 = (int64_t)42;`, `double _v2 = 3.14;`, `bool _v3 = true;`, `Iron_String _v4 = iron_string_from_literal("hello");`, `(void*)NULL` for null
  - **Arithmetic/comparison/logical**: `T _vN = _vL op _vR;` with op being +,-,*,/,%, ==,!=,<,<=,>,>=, &&,||
  - **Unary**: `T _vN = -_vO;` or `bool _vN = !_vO;`
  - **Alloca**: `T _vN;` (declare C variable of alloc_type)
  - **Load**: `T _vN = _vM;` (copy from alloca variable)
  - **Store**: `_vM = _vN;` (write into alloca variable)
  - **Get/Set field**: `T _vN = _vO.field;` / `_vO.field = _vN;` (or `->field` for pointers)
  - **Get/Set index**: port exact pattern from gen_exprs.c (uses iron_list_get/iron_list_set or array indexing)
  - **Call**: `T _vN = func_name(_vA, _vB);` for direct calls; indirect calls use function pointer cast. Void calls omit result.
  - **Jump**: `goto <target_label>;`
  - **Branch**: `if (_vC) goto <then_label>; else goto <else_label>;`
  - **Switch**: `switch (_vS) { case 0: goto L0; case 1: goto L1; ... default: goto Ld; }`
  - **Return**: `return _vN;` or `return;` for void
  - **Cast**: `T _vN = (T)_vM;`
  - **Heap alloc**: `T* _vN = malloc(sizeof(T)); ... ` (auto_free inserts corresponding free via IRON_IR_FREE)
  - **RC alloc**: `T* _vN = iron_rc_alloc(sizeof(T));`
  - **Free**: `free(_vM);`
  - **Construct**: `T _vN = { .field0=_v1, .field1=_v2 };` (field names from type->object.decl->fields[])
  - **Array lit**: Port exact pattern from gen_exprs.c IRON_NODE_ARRAY_LIT (iron_list_create + push)
  - **Slice**: Port from gen_exprs.c IRON_NODE_SLICE
  - **Is null/Is not null**: `bool _vN = _vM.has_value == false;` / `bool _vN = _vM.has_value == true;`
  - **Interp string**: Port exact snprintf two-pass pattern from gen_exprs.c IRON_NODE_INTERP_STRING
  - **Make closure**: Allocate env struct, populate captures, store function pointer
  - **Func ref**: `void* _vN = (void*)func_name;`
  - **Spawn**: `Iron_Future* _vN = iron_pool_submit(pool, lifted_func, env);`
  - **Parallel for**: `iron_pool_parallel_for(pool, range, chunk_func, env); iron_pool_barrier(pool);`
  - **Await**: `iron_future_await(_vM);`
  - **Poison**: `/* POISON */` (skip, should not appear in valid IR)
  - **Phi**: `assert(false && "phi should be eliminated before emission");`

**7. Main wrapper emission**:
- Detect `Iron_main` function in module->funcs[]. If found, emit:
```c
int main(int argc, char** argv) {
    (void)argc; (void)argv;
    iron_runtime_init();
    Iron_main();
    iron_runtime_shutdown();
    return 0;
}
```

**8. Lifted function handling**:
- Functions whose names contain "lambda_", "spawn_", or "parallel_" are lifted functions. Emit their bodies to the `lifted_funcs` section buffer instead of `implementations` so they appear before their use sites.

**9. Section concatenation** (same order as old codegen lines 529-553):
- Concatenate all sections into final output. Arena-strdup the result. Free all StrBufs.

**Critical notes from research:**
- Value naming: `_v{id}` scheme (e.g., `_v0`, `_v1`, `_v2`). Use `iron_strbuf_appendf(sb, "_v%u", id)`.
- Block labels: use `block->label` directly (already unique within function). C label syntax: `label:;\n`
- Reuse `iron_mangle_name()` and `iron_mangle_method()` from codegen.h for name consistency
- Port `topo_visit()` from codegen.c for struct ordering
- Emit `#define IRON_TAG_<mangled_name> <tag>` for each object struct (sequential from 1)
- For `iron_type_to_c()`: since it requires `Iron_Codegen*`, either create a thin wrapper that only uses the arena field, or replicate the type mapping. Recommendation: create a static `emit_type_to_c(const Iron_Type *t, Iron_Arena *arena)` that does the same mapping.
