Create `src/lir/value_range.h` with:

```c
typedef struct {
    int64_t min;
    int64_t max;
    bool is_top;  // true = unknown range, no compression
} ValueRange;

typedef struct {
    // Per-field ranges: key = "type_name:field_name" -> ValueRange
    struct { char *key; ValueRange value; } *field_ranges;
    Iron_Arena *arena;
} ValueRangeAnalysis;
```

Public API (3 functions, same pattern as layout_analysis):
- `void iron_vr_analyze(ValueRangeAnalysis *vra, IronLIR_Module *module, Iron_IfaceRegistry *iface_reg)` -- main entry point
- `const char *iron_vr_get_narrowed_type(ValueRangeAnalysis *vra, const char *type_name, const char *field_name)` -- returns narrower C type string or NULL
- `void iron_vr_free(ValueRangeAnalysis *vra)` -- releases internal hash map resources

Create `src/lir/value_range.c` with:

1. **Type ladder function** `select_narrowed_type(int64_t min, int64_t max)`:
   - min >= 0: UINT8_MAX -> "uint8_t", UINT16_MAX -> "uint16_t", UINT32_MAX -> "uint32_t", else NULL
   - min < 0: INT8_MIN/MAX -> "int8_t", INT16_MIN/MAX -> "int16_t", INT32_MIN/MAX -> "int32_t", else NULL
   - Returns NULL when range equals or exceeds 64-bit (no savings)

2. **Range arithmetic helpers** with overflow protection:
   - `range_union(a, b)`: `[min(a.min, b.min), max(a.max, b.max)]`, if either is_top -> TOP
   - `range_add(a, b)`: check `__builtin_add_overflow` for min+min and max+max, return TOP on overflow
   - `range_sub(a, b)`: `[a.min - b.max, a.max - b.min]` with overflow check -> TOP
   - `range_mul(a, b)`: compute all 4 products `{a.min*b.min, a.min*b.max, a.max*b.min, a.max*b.max}`, take min/max, overflow check -> TOP

3. **Per-function value range tracking** `analyze_function_ranges(vra, fn)`:
   - Build per-value range map: `struct { IronLIR_ValueId key; ValueRange value; } *value_ranges`
   - `IRON_LIR_CONST_INT`: range = `[value, value]`
   - `IRON_LIR_ADD/SUB/MUL`: apply range arithmetic on operands
   - `IRON_LIR_PHI`: union of all incoming value ranges
   - `IRON_LIR_LOAD`: look up most recent STORE range if tracked, else TOP
   - `IRON_LIR_CALL`: TOP (conservative -- values through function calls are unknown for now)
   - All other instructions producing integer values: TOP

4. **Field range collection** -- after computing per-value ranges:
   - Scan for `IRON_LIR_SET_FIELD`: look up the value's range and union into field_ranges[type_name:field_name]
   - Scan for `IRON_LIR_CONSTRUCT`: for each field_val in declaration order, look up its range and union into field_ranges using the construct type's field names from `construct.type->object.fields`
   - If a field is set from a value with is_top=true, the field range becomes TOP permanently

5. **Interprocedural analysis** (scan all functions):
   - For each function in module->funcs, run analyze_function_ranges
   - field_ranges accumulates across all functions (union semantics)
   - After all functions processed, any field_name that was never seen stays absent from the map (meaning all values came from constructor/set_field with known ranges)

6. **Query function** `iron_vr_get_narrowed_type()`:
   - Build key "type_name:field_name", look up in field_ranges
   - If not found: no range data -> return NULL (conservative, no compression)
   - If found and is_top: return NULL (unknown range)
   - If found with valid range: call select_narrowed_type(min, max), return result

7. **Free function** `iron_vr_free()`: `shfree(vra->field_ranges)`
