The auto-static method dispatch in gen_exprs.c emits `Iron_timer_<method>(args...)` passing arguments BY VALUE (no address-of operator). But the existing C functions `Iron_timer_since` and `Iron_timer_reset` take `const Iron_Timer *` and `Iron_Timer *` respectively. We need value-based wrapper functions.

**A. src/stdlib/iron_time.h** -- Add value-based wrappers after the existing Timer declarations (after line 17):
```c
/* Value-based wrappers for Iron auto-static dispatch (no pointer args) */
static inline int64_t Iron_timer_since_val(Iron_Timer t) {
    return Iron_time_now_ms() - t.start_ms;
}
static inline Iron_Timer Iron_timer_reset_val(Iron_Timer t) {
    t.start_ms = Iron_time_now_ms();
    return t;
}
```
Use `static inline` in the header since these are trivial one-liners. `Iron_timer_reset_val` returns the modified Timer by value (since the auto-static path cannot mutate through a pointer).

**B. src/stdlib/iron_time.c** -- No changes needed. The pointer-based originals remain for any future C callers.

**C. src/stdlib/time.iron** -- Add Timer object and its 3 static methods after the existing Time methods:
```
object Timer {
  val start_ms: Int
}

func Timer.create() -> Timer {}
func Timer.since(t: Timer) -> Int {}
func Timer.reset(t: Timer) -> Timer {}
```

The auto-static dispatch will map:
- `Timer.create()` -> `Iron_timer_create()` (already correct, returns Iron_Timer by value)
- `Timer.since(t)` -> `Iron_timer_since(t)` (PROBLEM: pointer signature)
- `Timer.reset(t)` -> `Iron_timer_reset(t)` (PROBLEM: pointer signature)

To fix the dispatch mismatch, we need to rename the value wrappers to match what codegen emits. The codegen emits `Iron_timer_since(t)` and `Iron_timer_reset(t)`. Since we cannot have two functions with the same name but different signatures in C, we must REPLACE the original pointer-based functions with value-based ones.

**REVISED approach A. src/stdlib/iron_time.h** -- Change the Timer function signatures to accept by value:
```c
/* ── Timer ───────────────────────────────────────────────────────────────── */
typedef struct { int64_t start_ms; } Iron_Timer;

Iron_Timer Iron_timer_create(void);
int64_t    Iron_timer_since(Iron_Timer t);    /* was: const Iron_Timer *t */
Iron_Timer Iron_timer_reset(Iron_Timer t);    /* was: Iron_Timer *t; now returns new Timer */
```

**REVISED approach B. src/stdlib/iron_time.c** -- Update implementations to match:
```c
int64_t Iron_timer_since(Iron_Timer t) {
    return Iron_time_now_ms() - t.start_ms;
}

Iron_Timer Iron_timer_reset(Iron_Timer t) {
    t.start_ms = Iron_time_now_ms();
    return t;
}
```
`Iron_timer_create` already works by value -- no change needed.

Note: `Iron_timer_reset` now returns a new Timer by value. This is the correct approach because:
1. The auto-static codegen passes args by value (no `&`)
2. Iron's `val` semantics mean Timer values are immutable -- users would write `val t2 = Timer.reset(t)` or use `var t` for reassignment
