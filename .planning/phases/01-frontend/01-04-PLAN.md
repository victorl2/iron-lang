1. Update src/parser/parser.c — String Interpolation Parsing:
   - Add iron_parse_interp_string(Iron_Parser *p, const char *raw_value, Iron_Span span) function.
   - When the parser encounters an IRON_TOK_INTERP_STRING token (tagged by lexer for strings containing {):
     * Split the raw string value on { and } boundaries.
     * For each literal segment: create an Iron_StringLit node.
     * For each {expr} segment: create a mini-lexer on the expression text, lex it, parse it as an expression using iron_parse_expr_prec. This creates the correct expression AST node.
     * Alternate these parts in an Iron_InterpString.parts stb_ds array.
     * Example: "Hello {name}, you have {hp} HP" becomes:
       InterpString.parts = [StringLit("Hello "), Ident("name"), StringLit(", you have "), Ident("hp"), StringLit(" HP")]
   - If a {expr} segment fails to parse: emit E0102 diagnostic, insert ErrorNode as that segment.
   - IMPORTANT: The InterpolatedString parsing re-uses the existing expression parser. Do NOT write a separate expression parser.

2. Update src/parser/parser.c — Harden Error Recovery:
   - Ensure iron_parser_sync_toplevel and iron_parser_sync_stmt are called correctly after every error.
   - Add in_error_recovery flag to Iron_Parser struct (if not already present from Plan 03).
   - After inserting an ErrorNode and calling sync: set p->in_error_recovery = true.
   - In iron_parser_sync_stmt: advance past tokens until hitting a statement-starting keyword (val, var, if, for, while, match, return, defer, free, leak, spawn, func, object), or }, or EOF.
   - After successful parse of next valid construct: set p->in_error_recovery = false.
   - CRITICAL: When in_error_recovery == true, suppress additional diagnostic emission. This prevents one bad token from generating N follow-on errors. Only emit the first error before sync, suppress until recovery succeeds.
   - Test behavior: a file with 3 independent syntax errors (each in a separate declaration or separate statement with valid code between them) produces exactly 3 diagnostics.
