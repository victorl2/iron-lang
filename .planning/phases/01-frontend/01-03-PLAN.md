4. Create src/parser/parser.c implementing complete recursive descent:

   a. Helper functions:
      - static Iron_Token *iron_current(Iron_Parser *p): return &p->tokens[p->pos]
      - static Iron_TokenKind iron_peek(Iron_Parser *p): return current token kind
      - static Iron_Token *iron_advance(Iron_Parser *p): return current and advance pos; skip NEWLINE tokens automatically
      - static bool iron_check(Iron_Parser *p, Iron_TokenKind kind): peek == kind
      - static bool iron_match(Iron_Parser *p, Iron_TokenKind kind): if check, advance, return true
      - static Iron_Token *iron_expect(Iron_Parser *p, Iron_TokenKind kind): if !check, emit diagnostic with appropriate E-code, return NULL; else advance and return token
      - static void iron_skip_newlines(Iron_Parser *p): while peek == NEWLINE, advance
      - static Iron_Span iron_token_span(Iron_Token *t): create span from token line/col/len
      - static Iron_Node *iron_make_error(Iron_Parser *p): create ErrorNode at current span
      - static void iron_parser_sync_toplevel(Iron_Parser *p): advance until func/object/interface/enum/import/EOF
      - static void iron_parser_sync_stmt(Iron_Parser *p): advance until keyword that starts a statement, or }, or EOF

   b. Top-level parsing:
      - iron_parse: create Program node, loop parsing declarations until EOF:
        * Skip newlines
        * If peek == IRON_TOK_PRIVATE: mark next decl as private
        * If peek == IRON_TOK_FUNC: parse iron_parse_func_or_method
        * If peek == IRON_TOK_OBJECT: parse iron_parse_object_decl
        * If peek == IRON_TOK_INTERFACE: parse iron_parse_interface_decl
        * If peek == IRON_TOK_ENUM: parse iron_parse_enum_decl
        * If peek == IRON_TOK_IMPORT: parse iron_parse_import_decl
        * If peek == IRON_TOK_VAL: parse iron_parse_val_decl
        * If peek == IRON_TOK_VAR: parse iron_parse_var_decl
        * Otherwise: emit error, sync to top-level, insert ErrorNode

      - iron_parse_func_or_method: parse "func", then identifier. If next is ".", it's a method (Type.method). Parse param list in parens, optional "-> Type" return, parse block body.
      - iron_parse_object_decl: parse "object Name", optional "[T]" generics, optional "extends Parent", optional "implements I1, I2", parse { fields }.
      - iron_parse_interface_decl: parse "interface Name { method_signatures }".
      - iron_parse_enum_decl: parse "enum Name { VARIANT1, VARIANT2, }".
      - iron_parse_import_decl: parse "import path.to.module" with optional "as alias".

   c. Statement parsing (iron_parse_stmt):
      - val/var declarations: parse "val/var name [: Type] = expr"
      - Assignment: parsed after expr; if expr followed by =, +=, -=, *=, /=, reinterpret as assignment target
      - return: parse "return [expr]"
      - if/elif/else: parse "if cond { body } [elif cond { body }]* [else { body }]"
      - while: parse "while cond { body }"
      - for: parse "for name in expr [parallel [(pool)]] { body }"
      - match: parse "match expr { Pattern { body } ... [else { body }] }"
      - defer: parse "defer expr"
      - free: parse "free expr"
      - leak: parse "leak expr"
      - spawn: parse "spawn(\"name\" [, pool]) { body }" with optional "val handle ="
      - block: parse "{ stmt* }"
      - Otherwise: parse as expression statement

   d. Expression parsing using Pratt parser (iron_parse_expr with precedence parameter):

      Precedence levels (as ints, lower = lower precedence):
      PREC_NONE = 0, PREC_ASSIGN = 1, PREC_IS = 2, PREC_OR = 3, PREC_AND = 4,
      PREC_EQUALITY = 5, PREC_COMPARISON = 6, PREC_TERM = 7, PREC_FACTOR = 8,
      PREC_UNARY = 9, PREC_CALL = 10

      - iron_parse_expr_prec(p, min_precedence): Pratt parser main loop
        * Parse prefix: literal, identifier, unary (-, not), grouping (parens), heap, rc, comptime, await, lambda (func(...) {}), array literal
        * Loop while current token has higher precedence than min_precedence:
          * Binary ops: +, -, *, /, %, ==, !=, <, >, <=, >=, and, or
          * Postfix: . (field access or method call), [ (index or slice), ( (function call)
          * is: type check expression
        * Return the expression tree

      - iron_parse_expr(p): calls iron_parse_expr_prec(p, PREC_ASSIGN + 1) (assignment handled at statement level)

      - Specific expression parsers:
        * iron_parse_call_args: parse "(" [expr ("," expr)*] ")"
        * iron_parse_lambda: parse "func" "(" params ")" ["->" type] "{" body "}"
        * iron_parse_type_annotation: parse type name with optional "?", optional "[T]" generics, optional array "[T; N]"

   e. Generic parameter parsing:
      - iron_parse_generic_params: parse "[T, U, ...]" after func/object names

5. CRITICAL: Every node constructor must set span and kind as the first two operations.
   All node pointers returned from parse functions must be non-NULL — use iron_make_error() on failure.
