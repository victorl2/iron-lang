2. Create src/lexer/lexer.c implementing the complete lexer:

   a. Keyword table: sorted array of {const char *name, Iron_TokenKind kind} pairs for all 37 keywords. Use bsearch for O(log n) lookup. Keywords: and, await, comptime, defer, elif, else, enum, extends, false, for, free, func, heap, if, implements, import, in, interface, is, leak, match, not, null, object, or, parallel, pool, private, rc, return, self, spawn, super, true, val, var, while

   b. Helper functions:
      - static char iron_peek_char(Iron_Lexer *l): return current char or '\0' if at end
      - static char iron_peek_next(Iron_Lexer *l): return next char or '\0'
      - static char iron_advance_char(Iron_Lexer *l): return current and advance pos, update col
      - static void iron_skip_whitespace(Iron_Lexer *l): skip spaces and tabs only (NOT newlines)
      - static Iron_Token iron_make_token(Iron_Lexer *l, Iron_TokenKind kind, const char *value, uint32_t start_line, uint32_t start_col, uint32_t len): create token struct

   c. Scanning functions:
      - static Iron_Token iron_lex_string(Iron_Lexer *l):
        * Record start position. Advance past opening ".
        * Scan until closing " or end of line/file.
        * If '\n' or EOF before closing ": emit E0001 diagnostic ("unterminated string literal"), suggestion "add a closing `\"` at the end of the string", return ERROR token.
        * Track whether any { is found inside the string (for interpolation detection).
        * Handle multiline strings: if starts with """, scan until closing """, allow newlines.
        * Copy string content (between quotes) into arena via iron_arena_strdup.
        * If string contains {: return IRON_TOK_INTERP_STRING, else return IRON_TOK_STRING.
        * Handle escape sequences: \n, \t, \\, \", \{

      - static Iron_Token iron_lex_number(Iron_Lexer *l):
        * Scan digits. If '.' followed by digit, scan float. Return IRON_TOK_INTEGER or IRON_TOK_FLOAT.
        * Copy numeric text into arena.
        * If invalid suffix (letter after number), emit E0003 diagnostic.

      - static Iron_Token iron_lex_identifier(Iron_Lexer *l):
        * Scan [a-zA-Z_][a-zA-Z0-9_]*
        * Copy into arena.
        * Look up in keyword table via bsearch. If found, return keyword token kind.
        * Otherwise return IRON_TOK_IDENTIFIER.

      - static Iron_Token iron_lex_punctuation(Iron_Lexer *l):
        * Handle multi-char operators: ==, !=, <=, >=, ->, .., +=, -=, *=, /=
        * Handle -- as comment: scan to end of line, skip, return IRON_TOK_NEWLINE (comment consumed)
        * Single-char operators: +, -, *, /, %, =, <, >, ., ,, :, ?, (, ), [, ], {, }, ;
        * If unrecognized character: emit E0002 diagnostic ("invalid character"), suggestion NULL, advance past it, return ERROR token.

   d. Main lex loop (iron_lex_all):
      * Create stb_ds token array (NULL initially, use arrput)
      * Loop: skip whitespace, check for newline/EOF/string/number/identifier/punctuation
      * For newlines: produce IRON_TOK_NEWLINE, advance line, reset col
      * For EOF: produce IRON_TOK_EOF, break
      * arrput each token
      * Return the array

   e. iron_token_kind_str: static const char* array indexed by token kind, return "IRON_TOK_VAL" etc.
