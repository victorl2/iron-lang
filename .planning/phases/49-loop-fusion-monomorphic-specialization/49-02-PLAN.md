        bool terminal_is_void = (strcmp(terminal->method, "forEach") == 0);
        bool terminal_is_collection = !terminal_is_scalar && !terminal_is_void;

    }

    **For flat array fused loop emission**, emit the following C code structure:

    1. **Declare result variable** (if not void/forEach):
       - For scalar terminal (reduce/sum): declare result with type from terminal_instr->type, initialize to reduce's init_arg value or 0 for sum
       - For collection terminal (map/filter): declare result list with `_create()`
       - For forEach: no result variable

    2. **Open scoped block** `{` with lambda typedefs and memcpy casts:
       - For each chain node that has lambda_args, emit a typedef for the lambda function pointer and memcpy cast, following the exact pattern from the existing split collection dispatch (typedef + memcpy from `.fn` field). Use unique names: `_FuseMapFn_0`, `_FuseFilterFn_1`, etc. using the node's position index.
       - Emit corresponding env pointers: `void *_fuse_env_0 = ` ... `.env;`

    3. **Emit the loop** `for (int64_t _fi = 0; _fi < {source}.count; _fi++)`:
       - Element extraction: `{elem_type} _fuse_elem = {source}.items[_fi];`
       - For each chain node, in order, emit the operation:
         - **map**: `{mapped_type} _fuse_v{pos} = _fuse_fn_{pos}(_fuse_env_{pos}, _fuse_prev);`
           where _fuse_prev is `_fuse_elem` for pos=0, or the previous map's result
         - **filter**: `if (!_fuse_fn_{pos}(_fuse_env_{pos}, _fuse_prev)) continue;`
         - **reduce**: `{result_var} = _fuse_fn_{pos}(_fuse_env_{pos}, {result_var}, _fuse_prev);`
         - **sum**: `{result_var} += _fuse_prev;`
         - **forEach**: `_fuse_fn_{pos}(_fuse_env_{pos}, _fuse_prev);`
       - For collection terminals (map that's the terminal, or filter):
         - **map terminal**: push `_fuse_v{pos}` to result list
         - **filter terminal**: push `_fuse_prev` (the element that passed) to result list

    4. **Close scoped block** `}`

    5. **Handle hoisted values**: If the terminal_instr was hoisted (in phi_hoisted), emit assignment-only form (no type prefix). Otherwise emit with type prefix for the result variable.

       e. For SoA types: access per-field arrays and reconstruct struct (follow Phase 48 patterns)
       f. Apply all chain operations inline (same as flat, but per-type)
       g. For sequential reduce/sum accumulation: the accumulator variable is shared across all type loops (circle loop runs first, square loop continues from same acc)
    4. For collection terminals that return a split collection: each per-type loop pushes to its own sub-array in the result split list. Track the result as a split collection in `ctx->split_collection_ids`.
