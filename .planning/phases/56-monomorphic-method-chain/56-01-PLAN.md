<objective>
Fix the root cause of the monomorphic-collapsed collection + method chain bug: generated C references `Iron_List_Iron_<Type>` symbols (struct, push, get, set, len, pop, free) that are never declared because `IRON_LIST_DECL`/`IRON_LIST_IMPL` macros are only pre-instantiated for primitives (`int64_t`, `int32_t`, `double`, `bool`, `Iron_String`, `Iron_Closure`) at `iron_runtime.h:640-645`. After Phase 49 mono collapse removes a split collection from `ctx->split_collection_ids`, the plain-typed-array emission path runs for concrete object types without the required list type decls existing.

Phase 56 Plan 01 adds a pre-scan pass in `emit_structs.c` that iterates `ctx->monomorphic_collections`, collects concrete object type names, mangles them via `emit_mangle_name`, dedups through a new `stb_ds` hash set `emitted_mono_list_types`, and emits `IRON_LIST_DECL(Iron_<T>, Iron_<T>)` + `IRON_LIST_IMPL(Iron_<T>, Iron_<T>)` into `ctx->struct_bodies` before any function body. This fixes both symptom paths in one surgical change:
1. True mono chain: `val shapes = [Circle(3), Circle(4)]; shapes.map(f).sum()`
2. Single-element narrowing: `var shapes = [Circle(1)]; shapes.push(Circle(2))` — same mono collapse path.

Plan 01 also ships the full positive-coverage sweep: primary regression test `mono_method_chain.iron` (ROADMAP SC4), per-method parity tests for all 10 stdlib collection methods, chain combinations (2-op and 3-op), a fusion probe test, and an adjacent test with two different concrete types. If the fusion probe reveals `emit_fusion.c` doesn't handle the plain-typed-array path after mono collapse, the scope flexes to extend the fusion engine in this plan.

**Step 1 — Declare the static helper at the top of emit_structs.c (after the existing `static bool ir_has_subtype(...)` at line 82):**

```c
/* ── Phase 56: Monomorphic list type decl emission ──────────────────────
 * For each concrete object type that appears in ctx->monomorphic_collections
 * (populated by the Phase 49/53 mono detection scan in emit_c.c), emit the
 * Iron_List_<mangled> struct typedef + IRON_LIST_DECL + IRON_LIST_IMPL macros
 * into ctx->struct_bodies. Dedups via stb_ds hash set emitted_mono_list_types.
 *
 * Fixes the bug where Phase 49 mono collapse removes collections from
 * split_collection_ids, causing generated C to reference Iron_List_Iron_Circle
 * etc. without those symbols being declared. */
static void emit_mono_list_decls(EmitCtx *ctx) {
    if (!ctx->monomorphic_collections) return;
    if (hmlen(ctx->monomorphic_collections) == 0) return;

    /* Dedup set: keyed by mangled concrete type name (e.g. "Iron_Circle").
     * Per-compilation-unit scope — freed at end of function. */
    struct { char *key; bool value; } *emitted_mono_list_types = NULL;

    /* Collect unique concrete type names. Value in monomorphic_collections is
     * BARE type name ("Circle") from emit_c.c:5353 et->object.decl->name. */
    for (ptrdiff_t i = 0; i < hmlen(ctx->monomorphic_collections); i++) {
        const char *bare_type = ctx->monomorphic_collections[i].value;
        if (!bare_type) continue;

        /* Mangle: "Circle" -> "Iron_Circle" (arena-allocated) */
        const char *mangled = emit_mangle_name(bare_type, ctx->arena);

        /* Dedup check */
        if (shgeti(emitted_mono_list_types, mangled) >= 0) continue;
        shput(emitted_mono_list_types, mangled, true);

        /* Emit Iron_List_<mangled> struct typedef. The IRON_LIST_DECL/IMPL
         * macros assume this struct is already declared with fields
         * { T *items; int64_t count; int64_t capacity; }. */
        iron_strbuf_appendf(&ctx->struct_bodies,
            "/* Phase 56: Iron_List type for mono-collapsed %s */\n"
            "typedef struct Iron_List_%s {\n"
            "    %s    *items;\n"
            "    int64_t count;\n"
            "    int64_t capacity;\n"
            "} Iron_List_%s;\n",
            mangled, mangled, mangled, mangled);

        /* Emit IRON_LIST_DECL(T, suffix) — function prototypes */
        iron_strbuf_appendf(&ctx->struct_bodies,
            "IRON_LIST_DECL(%s, %s)\n",
            mangled, mangled);

        /* Emit IRON_LIST_IMPL(T, suffix) — function bodies (non-static).
         * Safe at translation-unit level because each mangled name is unique
         * per compilation unit (Iron compiles to one .c file per program). */
        iron_strbuf_appendf(&ctx->struct_bodies,
            "IRON_LIST_IMPL(%s, %s)\n\n",
            mangled, mangled);
    }

    shfree(emitted_mono_list_types);
}
```

3. Expected output is deterministic and verifiable by byte-comparing stdout to .expected.

**Critical: how to force mono collapse.** Per the mono detection scan at emit_c.c:5281-5540, a collection is monomorphic if:
- It's created via `ARRAY_LIT` (not a CALL result — unless the called function has known return types)
- ALL elements are the same concrete object type (no mixed types, no `[Circle, Square]`)
- It does NOT escape via CALL arg, RETURN, SET_FIELD, or MAKE_CLOSURE

The simplest pattern: `val shapes = [Circle(1), Circle(2), Circle(3)]` — all Circle, no escape. The type checker assigns `[Circle]` (concrete) or `[Shape]` (interface) depending on context; either way, the mono detection sees one concrete type in the elements and marks it for collapse.

**Alternative that exercises the single-element narrowing path:** `var shapes = [Circle(1)]` — single-element literal also narrows to `[Circle]` and collapses.
