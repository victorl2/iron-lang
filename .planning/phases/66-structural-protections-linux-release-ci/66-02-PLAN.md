Enable `-Werror=switch-enum` globally across all non-MSVC targets (iron_compiler, iron_runtime, iron_stdlib, ironc) by appending the flag to the existing warning line in CMakeLists.txt, then audit every switch over an Iron enum in the affected sources (including the 6 WebAssembly files added by the post-merge PR #23) and either promote each switch to exhaustive case coverage OR annotate a surviving `default:` with a `/* -Wswitch-enum opt-out: <reason> */` comment. Purpose: Satisfies PROT-02. The opt-out comments leave a grep-able audit trail for future tooling.

Output: Modified CMakeLists.txt with the new flag, plus edits across ~18 source files that either add missing cases or add opt-out comments. `cmake --build build` must succeed cleanly with the new flag; full unit + integration test suite must pass.

**Step 5: Convert emit_c.c FFI cast if-chain to an exhaustive switch.**

```c
if (arg_type && arg_type->kind == IRON_TYPE_STRING) { is_string_arg = true; }
...
if ((arg_type->kind == IRON_TYPE_INT || arg_type->kind == IRON_TYPE_INT64) && ...) { needs_int_cast = true; }
```

This is PROT-02's "make if-chains into switches under strictness posture" scope. Rewrite as:
```c
switch (arg_type->kind) {
    case IRON_TYPE_STRING:
        is_string_arg = true;
        break;
    case IRON_TYPE_INT:
    case IRON_TYPE_INT64:
        if (pt && (pt->kind == IRON_TYPE_INT || pt->kind == IRON_TYPE_INT32)) {
            needs_int_cast = true;
        }
        break;
    case IRON_TYPE_FLOAT64:
        if (pt && pt->kind == IRON_TYPE_FLOAT32) {
            needs_float_cast = true;
        }
        break;
    default:
        /* -Wswitch-enum opt-out: FFI cast bridge only cares about STRING / INT /
         * INT64 / FLOAT64 argument types; all other kinds pass through unchanged
         * to the default emit path. */
        break;
}
```

Preserve existing semantics exactly (`is_string_arg`, `needs_int_cast`, `needs_float_cast` booleans computed identically). Verify by running the full test suite.
