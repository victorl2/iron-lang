    **src/cli/fmt.c** — Format command:
    1. Read the source file into a string
    2. Create arena and diag list
    3. Call `iron_lex(source, arena, diags)` to tokenize
    4. Call `iron_parse(tokens, arena, diags)` to get AST
    5. Check for parse errors. If errors exist:
       - Print diagnostics to stderr
       - Print "iron fmt: refusing to format file with syntax errors"
       - Return 1 (leave original file untouched — Pitfall 7)
    6. Call `iron_print_ast(program_node, arena)` to get formatted source
    7. Write formatted output to a temp file: `<original_path>.iron.tmp`
    8. Verify temp file was written successfully (check return of fwrite)
    9. Atomically rename temp file to original: `rename(tmp_path, source_path)`
    10. Free arena
    11. Return 0

    The key safety rule: NEVER write to the original file until the parse succeeds and the formatted output is fully written to temp. If anything fails, the original is untouched.

    Include: `<stdio.h>`, `<stdlib.h>`, `<string.h>`, relevant compiler headers.
  </action>

    **src/cli/test_runner.c** — Test discovery and runner:
    1. If dir_path is NULL, use current directory "."
    2. Open directory with `opendir(dir_path)`
    3. Iterate with `readdir()`, collect all files matching `test_*.iron`
    4. Sort the file list alphabetically for deterministic order
    5. For each test file:
       a. Print `[RUN ] <filename>`
       b. Compile with `iron_build(filepath, tmp_binary, {.run_after=false})` to create binary
       c. If compilation fails: print `[FAIL] <filename> (compilation error)`, increment fail count
       d. Execute the binary via `fork()+execvp()`, capture exit code
       e. If exit code 0: print `[PASS] <filename>`, increment pass count
       f. If exit code != 0: print `[FAIL] <filename> (exit code N)`, increment fail count
       g. Remove temp binary
    6. Print summary: `\nResults: <pass> passed, <fail> failed, <total> total`
    7. Return 0 if all passed, 1 if any failed

    Color output: `[PASS]` in green, `[FAIL]` in red, `[RUN ]` in blue — using ANSI codes gated by `isatty(STDOUT_FILENO)`.

    **src/cli/toml.h**:
    ```c
    #ifndef IRON_CLI_TOML_H
    #define IRON_CLI_TOML_H

    typedef struct {
        char *name;        // [project] name
        char *version;     // [project] version
        char *entry;       // [project] entry
        bool raylib;       // [dependencies] raylib = true
    } IronProject;

    /* Parse iron.toml at the given path. Returns NULL on error. */
    IronProject *iron_toml_parse(const char *path);
    void iron_toml_free(IronProject *proj);
    #endif
    ```

    **src/cli/toml.c** — Minimal TOML parser:
    Hand-written parser for the specific iron.toml format. Only supports:
    - `[project]` section with `name`, `version`, `entry` string values
    - `[dependencies]` section with `raylib = true/false`
    - Comments: `#` to end of line
    - String values: `"quoted"` or bare words

    Implementation:
    1. Read file into buffer
    2. Parse line by line
    3. Track current section (`[project]`, `[dependencies]`)
    4. For each `key = value` line in current section, assign to IronProject fields
    5. Return allocated IronProject (caller frees with iron_toml_free)
    6. If file doesn't exist or can't parse, return NULL

    This is intentionally minimal — only the fields we need. Not a full TOML parser.
