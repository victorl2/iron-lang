# Phase 3 Plan 03: Generic Collections (List, Map, Set) Summary

**One-liner:** Macro-generated List[T], Map[K,V], Set[T] collections with names exactly matching gen_types.c monomorphization convention (Iron_List_int64_t_push), backed by 18 Unity tests, all passing under ASan.

## What Was Built

### Task 1: Collection macros and common type instantiation

Added three macro families to `iron_runtime.h`:

- `IRON_LIST_DECL(T, suffix)` / `IRON_LIST_IMPL(T, suffix)` — dynamic array with push, get, set, pop, len, free, create
- `IRON_MAP_DECL(K, V, ksuffix, vsuffix)` / `IRON_MAP_IMPL(K, V, ksuffix, vsuffix, eq_fn)` — array-based map with put, get, has, remove, len, free, create
- `IRON_SET_DECL(T, suffix)` / `IRON_SET_IMPL(T, suffix, eq_fn)` — array-based set with add, contains, remove, len, free, create

Function names generated by the macros match `gen_types.c::mangle_generic()` exactly:
- `Iron_List_int64_t_push(Iron_List_int64_t *self, int64_t item)`
- `Iron_Map_Iron_String_int64_t_put(...)`
- `Iron_Set_int64_t_add(...)`

Created `iron_collections.c` with IMPL expansions for:
- List: int64_t, double, bool, Iron_String
- Map: Iron_String->int64_t, Iron_String->Iron_String
- Set: int64_t, Iron_String

### Task 2: Unity test suite

Created `tests/test_runtime_collections.c` with 18 tests across all three collection types:
- 8 List tests (create, push/get, len, set, pop, grow-to-100, string elements, free)
- 5 Map tests (put/get, has, update/overwrite, remove, len)
- 3 Set tests (add/contains, no-duplicates, remove)
- 2 additional: Map[String,String] and Set[String] variant coverage

## Decisions Made

1. **eq_fn parameter in Map/Set macros:** Rather than hardcoding equality per type, the macros accept an `eq_fn(const T *a, const T *b) -> bool` parameter. This enables type-safe pointer-based equality without requiring the macro to know the element type's equality semantics. Two helpers are defined in `iron_collections.c`: `iron_string_eq_ptr` and `int64_eq_ptr`.

2. **IRON_CODEGEN_PROVIDES_STRUCTS guard:** Pre-instantiated struct typedefs live in `iron_runtime.h` under this guard (defined as absent by default). When the codegen includes its own typedefs in generated C, it can define `IRON_CODEGEN_PROVIDES_STRUCTS` before including the runtime header to suppress redefinition. C11 also permits duplicate compatible typedefs so this is a belt-and-suspenders approach.

3. **O(n) linear scan:** For v1, map and set use linear scan. The API surface (put/get/has/remove/len/free) is compatible with a future hash table upgrade.
