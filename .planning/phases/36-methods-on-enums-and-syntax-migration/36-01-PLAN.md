Apply exactly four changes. Each is a small additive `else if` or guard relaxation. No new code paths, no new functions.

**Change 1: `src/analyzer/resolve.c` line 176 — attach_method guard**

Change the guard in `attach_method()` from:
```c
if (owner->sym_kind != IRON_SYM_TYPE) {
```
to:
```c
if (owner->sym_kind != IRON_SYM_TYPE && owner->sym_kind != IRON_SYM_ENUM) {
```

This is the root enabler. Without it, all enum methods emit "method declared on non-object type" and `md->owner_sym` stays NULL, blocking all downstream processing. The rest of the attach_method function (setting `md->owner_sym = owner`) works unchanged because enum symbols already carry `owner->type` set to `iron_type_make_enum(...)` from resolver pass 1a.

**Change 2: `src/hir/hir_lower.c` around line 1354 — self-type lookup in HIR pass 1**

After the existing `IRON_NODE_OBJECT_DECL` block (lines 1348-1354), add an `else if` branch for `IRON_NODE_ENUM_DECL`. The current loop only matches object declarations to find the self type:

```c
if (d->kind == IRON_NODE_OBJECT_DECL) {
    Iron_ObjectDecl *od = (Iron_ObjectDecl *)d;
    if (strcmp(od->name, md->type_name) == 0) {
        self_type = iron_type_make_object(mod->arena, od);
        break;
    }
}
```

Add after the closing `}` of that `if` block:
```c
else if (d->kind == IRON_NODE_ENUM_DECL) {
    Iron_EnumDecl *ed = (Iron_EnumDecl *)d;
    if (strcmp(ed->name, md->type_name) == 0) {
        self_type = iron_type_make_enum(mod->arena, ed);
        break;
    }
}
```

This ensures `params[0].type` (the self parameter) gets `IRON_TYPE_ENUM` for enum methods. Without this, `self_type` stays NULL and the C emitter will produce invalid code or crash.

**Change 3: `src/hir/hir_to_lir.c` around line 791 — type-name extraction for mangling**

After the existing `IRON_TYPE_OBJECT` check (lines 789-791):
```c
if (obj_type->kind == IRON_TYPE_OBJECT && obj_type->object.decl) {
    type_name = obj_type->object.decl->name;
}
```

Add an `else if` branch:
```c
else if (obj_type->kind == IRON_TYPE_ENUM && obj_type->enu.decl) {
    type_name = obj_type->enu.decl->name;
}
```

Without this, `type_name` stays as `"Unknown"`, and the call site emits `Iron_Unknown_area(self)` while the definition emits `Iron_shape_area(Iron_Shape self)`, causing a C linker error.

**Change 4: `src/analyzer/typecheck.c` around line 712 — method call return type for enum receivers**

After the existing `IRON_TYPE_OBJECT` check (lines 708-712):
```c
} else if (obj_id->resolved_type &&
           obj_id->resolved_type->kind == IRON_TYPE_OBJECT) {
    /* Instance method: receiver has object type */
    type_name_mc = obj_id->resolved_type->object.decl->name;
}
```

Add an `else if` branch:
```c
else if (obj_id->resolved_type &&
         obj_id->resolved_type->kind == IRON_TYPE_ENUM &&
         obj_id->resolved_type->enu.decl) {
    /* Instance method on enum value */
    type_name_mc = obj_id->resolved_type->enu.decl->name;
}
```

Without this, `type_name_mc` stays NULL, the method decl lookup is skipped, and `mc->resolved_type` defaults to `IRON_TYPE_VOID`. The return value is lost.
