<objective>
Create the raylib.iron wrapper file with extern func declarations and a Key enum using explicit ordinal values matching raylib KEY_* constants, extend enum codegen to support explicit values, extend the build pipeline to compile raylib source inline when iron.toml has raylib=true, and add --force-comptime flag.

Purpose: This enables Iron programs to use raylib for game development with the idiomatic dot-syntax for key codes (`is_key_down(.RIGHT)`) as locked in the user's decisions. The wrapper file provides the Iron-side API, enum codegen supports explicit values so Key variants map to correct raylib integer constants, and the build pipeline handles compiling raylib alongside the generated C for standalone binaries (GAME-03).

Output: raylib.iron wrapper with Key enum (explicit ordinals), window/drawing/input/audio extern declarations, extended AST/parser/codegen for explicit enum values, build.c extended for raylib compilation and --force-comptime, integration tests verifying extern func calling and draw block codegen.

  <action>
The Key enum in raylib.iron must have variants whose C integer values match raylib's KEY_* constants (e.g., KEY_RIGHT = 262). The current enum system emits 0, 1, 2... auto-incrementing C enum values. Extend the system to support explicit ordinal values.

1. **src/parser/ast.h** — Extend `Iron_EnumVariant` with optional explicit value:
   ```c
   typedef struct {
       Iron_Span     span;
       Iron_NodeKind kind;  /* IRON_NODE_ENUM_VARIANT */
       const char   *name;
       bool          has_explicit_value;
       int           explicit_value;   /* only valid when has_explicit_value */
   } Iron_EnumVariant;
   ```
   Initialize `has_explicit_value = false` and `explicit_value = 0` by default so all existing enums continue to work unchanged.

2. **src/parser/parser.c** — In the function that parses enum variants, after parsing the variant name check for `= <integer_literal>`:
   - If the next token is `=` followed by an integer literal, consume both tokens, set `ev->has_explicit_value = true` and `ev->explicit_value = (int)literal_value`.
   - Otherwise, leave `has_explicit_value = false` (auto-increment behavior preserved).
   - Syntax: `enum Key { RIGHT = 262, LEFT = 263, UP = 265, DOWN = 264, ... }`

3. **src/codegen/codegen.c** — In the enum definition emission loop (step 4, around line 339), emit explicit value when present:
   ```c
   for (int j = 0; j < ed->variant_count; j++) {
       Iron_EnumVariant *ev = (Iron_EnumVariant *)ed->variants[j];
       if (ev->has_explicit_value) {
           iron_strbuf_appendf(&ctx.enum_defs, "    %s_%s = %d",
                               mangled, ev->name, ev->explicit_value);
       } else {
           iron_strbuf_appendf(&ctx.enum_defs, "    %s_%s",
                               mangled, ev->name);
       }
       if (j < ed->variant_count - 1)
           iron_strbuf_appendf(&ctx.enum_defs, ",");
       iron_strbuf_appendf(&ctx.enum_defs, "\n");
   }
   ```
   This emits `Iron_Key_RIGHT = 262,` in the C typedef enum, which makes dot-syntax calls like `is_key_down(.RIGHT)` pass the correct raylib integer to C.

   Note: the dot-syntax `.RIGHT` resolves to the enum variant value through the existing enum variant access path in the resolver and codegen — no additional changes needed there beyond the explicit integer value being correct.

1. **Create src/stdlib/raylib.iron** — hand-written raylib bindings. Key enum MUST use explicit ordinal values matching raylib KEY_* constants so that `is_key_down(.RIGHT)` passes integer 262 to the C call. Function signatures for input take `Key`, not `Int`.

   ```iron
   -- raylib.iron — Iron bindings for raylib 5.5
   -- Ships with the Iron compiler; loaded when user writes: import raylib

   -- Types (match C struct layout exactly)
   object Vec2 { var x: Float32; var y: Float32 }
   object Color { var r: UInt8; var g: UInt8; var b: UInt8; var a: UInt8 }

   -- Color constants
   val RED      = Color(230, 41, 55, 255)
   val BLUE     = Color(0, 121, 241, 255)
   val GREEN    = Color(0, 228, 48, 255)
   val DARKGRAY = Color(80, 80, 80, 255)
   val WHITE    = Color(255, 255, 255, 255)
   val BLACK    = Color(0, 0, 0, 255)
   val YELLOW   = Color(253, 249, 0, 255)
   val ORANGE   = Color(255, 161, 0, 255)

   -- Key codes — explicit ordinal values match raylib KEY_* constants exactly.
   -- This is what makes is_key_down(.RIGHT) pass the correct integer to C.
   enum Key {
     SPACE     = 32,
     ESCAPE    = 256,
     ENTER     = 257,
     BACKSPACE = 259,
     RIGHT     = 262,
     LEFT      = 263,
     DOWN      = 264,
     UP        = 265,
     A = 65, B = 66, C = 67, D = 68, E = 69, F = 70,
     G = 71, H = 72, I = 73, J = 74, K = 75, L = 76,
     M = 77, N = 78, O = 79, P = 80, Q = 81, R = 82,
     S = 83, T = 84, U = 85, V = 86, W = 87, X = 88,
     Y = 89, Z = 90,
   }

   -- Window management
   extern func InitWindow(w: Int, h: Int, title: String)
   extern func CloseWindow()
   extern func WindowShouldClose() -> Bool
   extern func SetTargetFPS(fps: Int)
   extern func GetFrameTime() -> Float32

   -- Drawing
   extern func ClearBackground(color: Color)
   extern func BeginDrawing()
   extern func EndDrawing()
   extern func DrawText(text: String, x: Int, y: Int, size: Int, color: Color)
   extern func DrawRectangle(x: Int, y: Int, w: Int, h: Int, color: Color)
   extern func DrawCircle(x: Int, y: Int, radius: Float32, color: Color)
   extern func DrawLine(x1: Int, y1: Int, x2: Int, y2: Int, color: Color)

   -- Input — Key enum used here (not Int) per user decision
   extern func IsKeyDown(key: Key) -> Bool
   extern func IsKeyPressed(key: Key) -> Bool
   extern func GetMouseX() -> Int
   extern func GetMouseY() -> Int
   extern func IsMouseButtonPressed(button: Int) -> Bool
   ```

   NOTE on Vec2: Use Float32 (not Float) because raylib's Vector2 uses C `float`. Iron's Float maps to double which would cause wrong struct layout.

   NOTE on function names: extern func emits the Iron name as the C name without conversion. Raylib's API is CamelCase, so the Iron wrapper uses CamelCase names to match exactly. Users write `InitWindow(800, 600, "Game")` in their Iron code. This is intentional — it matches raylib's documented API surface.

2. **Import resolution for raylib** — how `import raylib` loads raylib.iron:
   In `iron_build()` in build.c, before running the pipeline, detect `import raylib` in the source. If found:
   - Locate `raylib.iron` at `{IRON_SOURCE_DIR}/stdlib/raylib.iron` (use the same make_src_path / compiler-relative path mechanism already used for the runtime).
   - Read the file contents.
   - Prepend the raylib.iron source to the user's source string, then run the normal lex/parse/analyze/codegen pipeline on the combined source.
   - The `import raylib` statement itself becomes a no-op (types and extern funcs are already in scope from the prepended source). This avoids multi-file parsing complexity.
   - Set `opts.use_raylib = true` when this path is taken so the build step links raylib.

3. **build.h** — Add two fields to `IronBuildOpts`:
   ```c
   bool use_raylib;       /* true when iron.toml has [dependencies] raylib = true */
   bool force_comptime;   /* --force-comptime flag: skip comptime cache */
   ```

4. **build.c** — Extend the build pipeline:
   - In `iron_build()`, check for iron.toml in the source directory. If `raylib = true` is parsed there, set `opts.use_raylib = true`.
   - Extend `invoke_clang()` (or create `invoke_clang_with_raylib()`) so that when `use_raylib` is true:
     - Adds `src/vendor/raylib/raylib.c` to the compilation (via make_src_path).
     - Adds `-DPLATFORM_DESKTOP` define.
     - Adds `-I` flag for the raylib header directory.
     - Adds platform-specific link flags:
       ```c
       #ifdef __APPLE__
       // -framework OpenGL -framework Cocoa -framework IOKit -framework CoreVideo
       #elif __linux__
       // -lGL -ldl -lrt
       #endif
       ```
   - Wire `--force-comptime` through to the build context (store in opts, pass to comptime integration from plan 02).

5. **main.c** — CLI argument parsing:
   - Add `--force-comptime` flag detection, setting `opts.force_comptime = true`.
   - Initialize `opts.use_raylib = false` and `opts.force_comptime = false` in default opts.
