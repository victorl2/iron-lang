  <action>
1. **comptime.c — read_file builtin:** In the `IRON_NODE_CALL` evaluation handler, when the callee name is `"read_file"` and arg_count == 1:
   - Evaluate the argument (must be a CVAL_STRING)
   - Construct the full path: concatenate `ctx->source_file_dir` + "/" + the string value
   - Open the file with fopen, read contents into arena-allocated buffer
   - If file doesn't exist, emit compile error: `"comptime read_file: cannot open '<path>'"` with the span from the call site
   - Return CVAL_STRING with the file contents
   - This counts as 1 step

   Also register "read_file" as a recognized comptime builtin. When the resolver doesn't find it (since it's not declared), the comptime evaluator handles it specially by name. Alternatively, register "read_file" in the global scope during resolver init as a builtin function with signature `func(String) -> String`.

   **Register in resolve.c:** Add `read_file` to the builtin registration block (alongside print, println, len, etc.) with type `func(String) -> String`:
   ```c
   /* read_file(String) -> String — comptime only */
   {
       Iron_Type *params[1] = { str_t };
       Iron_Type *fn = iron_type_make_func(arena, params, 1, str_t);
       Iron_Symbol *sym = iron_symbol_create(arena, "read_file",
                                              IRON_SYM_FUNCTION, NULL, no_span);
       sym->type = fn;
       iron_scope_define(ctx.global_scope, arena, sym);
   }
   ```

2. **comptime.c — FNV-1a hash function:**
   ```c
   static uint64_t fnv1a_hash(const char *data, size_t len) {
       uint64_t hash = 14695981039346656037ULL;
       for (size_t i = 0; i < len; i++) {
           hash ^= (uint8_t)data[i];
           hash *= 1099511628211ULL;
       }
       return hash;
   }
   ```

3. **comptime.c — cache read/write:**
   Add two static functions:
   - `comptime_cache_read(const char *source_text, size_t source_len, const char *func_name)`:
     - Compute FNV-1a hash of `source_text` (entire source file — conservative but correct per research)
     - Construct path: `.iron-build/comptime/<hex_hash>.cache`
     - If file exists, read it. Format: first line is result_kind (int), remaining lines are the value. For INT: `"0\n42"`, for STRING: `"3\nhello world"`, for FLOAT: `"1\n3.14"`, for BOOL: `"2\n1"`.
     - Return an `Iron_ComptimeVal*` or NULL if cache miss
   - `comptime_cache_write(const char *source_text, size_t source_len, const char *func_name, Iron_ComptimeVal *val)`:
     - Create `.iron-build/comptime/` directory if it doesn't exist (mkdir -p)
     - Compute hash, write the value to the cache file
     - For arrays, write count followed by each element value

4. **iron_comptime_apply — integrate cache:**
   Before evaluating each IRON_NODE_COMPTIME:
   - If `!force_comptime`, try cache read with the source file text
   - On cache hit, convert cached value to AST and replace the node
   - On cache miss (or force), evaluate normally, then write to cache

5. **comptime.h:** Add `const char *source_text; size_t source_len;` to `Iron_ComptimeCtx` for cache key computation.

6. **Wire source_file_dir in build.c:** In `iron_build()`, extract the directory from `source_path` using `dirname()` and pass it through to `iron_comptime_apply()` via the analysis result or directly. Currently `iron_analyze` doesn't take source_file_dir. Options:
   - Add `source_file_dir` and `force_comptime` parameters to `iron_analyze()`
   - Or call `iron_comptime_apply()` directly from `iron_build()` after `iron_analyze()` returns
   **Preferred:** Extend `iron_analyze` to accept `const char *source_file_dir` and `bool force_comptime`, passing them to `iron_comptime_apply` internally. Update `Iron_AnalyzeResult iron_analyze(...)` signature. Update all callers (build.c, check.c, test pipeline helpers) — most can pass NULL/false.
