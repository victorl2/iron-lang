<objective>
Hoist the array-for loop bound evaluation (`GET_FIELD .count`) from the header block to the pre-header block in `hir_to_lir.c`, so the bound is computed once per loop instead of on every back-edge iteration.

Purpose: Eliminates redundant struct field access on every loop iteration for all `for x in array` loops, the primary performance target identified in Phase 24 research (expected 1.3-1.5x speedup on for-range-heavy benchmarks).

Output: Modified `hir_to_lir.c` with hoisted bound, new unit test confirming placement, all tests and benchmarks green.
</objective>

**Step 2 — GREEN: Implement the hoisting fix**

In `src/hir/hir_to_lir.c`, in the `IRON_HIR_STMT_FOR` case (around line 1242, after `var_alloca` creation and before the `if (!block_is_terminated(pre_header))` jump):

Add these 3 lines (still inside the `switch_block(ctx, pre_header)` section):
```c
/* Hoist loop bound: evaluate .count once in pre-header */
IronLIR_ValueId count_alloca = emit_alloca_in_entry(ctx, int_type, "for_count", span);
IronLIR_ValueId count_init = iron_lir_get_field(ctx->current_func, pre_header,
                                                  iterable_val, "count", int_type, span)->id;
iron_lir_store(ctx->current_func, pre_header, count_alloca, count_init, span);
```

Then in the header section (around lines 1250-1252), replace the `count_val` GET_FIELD with a LOAD from the alloca:

Change:
```c
IronLIR_ValueId count_val = iron_lir_get_field(ctx->current_func, header,
                                                 iterable_val, "count", int_type, span)->id;
```

To:
```c
IronLIR_ValueId count_val = iron_lir_load(ctx->current_func, header, count_alloca, int_type, span)->id;
```

**CRITICAL:** The `count_alloca` variable must be declared before `switch_block(ctx, header)` is called, and the GET_FIELD + STORE must execute while `pre_header` is the active block. The LOAD in the header uses `count_alloca` which is in scope because it's in the same `case` block.
