  <action>
Replace the IRON_NODE_INTERP_STRING stub (line ~291-294 in gen_exprs.c) with full snprintf two-pass emission.

The Iron_InterpString node has `parts` (array of Iron_Node*) and `part_count`. Parts alternate between string literals (IRON_NODE_STRING_LIT) and expressions (any expression node). Each part has a `resolved_type` set by the type checker.

Implementation approach using a GNU statement expression `({ ... })`:

1. Declare a unique variable prefix using the codegen's lambda_counter (increment it): `int _interp_idx = ctx->lambda_counter++;`

2. Emit the opening of a statement expression: `iron_strbuf_appendf(sb, "({ ");`

3. **Build the format string at C-codegen time** by iterating over `parts`:
   - For `IRON_NODE_STRING_LIT` parts: append the literal's `value` to the format string (escape any `%` as `%%`)
   - For expression parts: inspect `resolved_type->kind`:
     - `IRON_TYPE_INT` -> append `%lld`
     - `IRON_TYPE_FLOAT` -> append `%g`
     - `IRON_TYPE_BOOL` -> append `%s`
     - `IRON_TYPE_STRING` -> append `%s`
     - Default (object/other) -> append `%s` (assuming to_string would be handled)

4. **Build the args list** by iterating over parts again (expression parts only):
   - `IRON_TYPE_INT` -> emit `(long long)(expr)`
   - `IRON_TYPE_FLOAT` -> emit `(expr)` (already double)
   - `IRON_TYPE_BOOL` -> emit `((expr) ? "true" : "false")`
   - `IRON_TYPE_STRING` -> emit `iron_string_cstr(&(expr))` — use address-of with parenthesized expression to handle rvalues via compound literal if needed. Actually, since iron_string_cstr takes `const Iron_String *`, and the expression might be an rvalue, emit it into a temp var first.

5. **Two-pass snprintf pattern** (emitted into the statement expression):
```c
({
    // For each String-typed expression part, emit a temp var to hold the Iron_String
    // so we can take its address for iron_string_cstr
    Iron_String _is0 = <string_expr>;

    char _ibuf[1024];
    int _in = snprintf(_ibuf, sizeof(_ibuf), "<format_string>", <args...>);
    Iron_String _ir;
    if (_in < 1024) {
        _ir = iron_string_from_cstr(_ibuf, (size_t)_in);
    } else {
        char *_ihp = (char*)malloc((size_t)_in + 1);
        snprintf(_ihp, (size_t)_in + 1, "<format_string>", <args...>);
        _ir = iron_string_from_cstr(_ihp, (size_t)_in);
        free(_ihp);
    }
    _ir;
})
```

Use the `_interp_idx` to make variable names unique: `_ibuf_0`, `_in_0`, `_ir_0`, `_is_0_N` for string temps.

6. For each expression part that is `IRON_TYPE_STRING`, emit a temporary variable declaration before the snprintf call:
```c
Iron_String _is_0_0 = <emit_expr for string part>;
```
Then in the snprintf args, use `iron_string_cstr(&_is_0_0)`.

For non-string expression parts, they can be emitted inline in the snprintf args since they produce C scalars.

Key implementation details:
- The format string is constructed at C code-gen time (compile time of the Iron compiler), NOT at C runtime. It's a string literal in the generated C.
- Escape `%` characters in literal parts by doubling them: `%` -> `%%`
- The `({...})` GNU statement expression is a GCC/Clang extension but is fine since Iron targets clang specifically.
- The result type of the statement expression is `Iron_String` (the final `_ir;` line).
