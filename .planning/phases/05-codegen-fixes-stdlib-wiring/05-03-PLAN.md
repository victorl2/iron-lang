**IMPORTANT NAMING INSIGHT**: The existing extern func mechanism uses `iron_snake_to_camel()` to convert Iron snake_case names to CamelCase C names. For example, `extern func init_window(...)` becomes `InitWindow(...)` in C. But the stdlib C functions are named like `Iron_math_sin` -- these are NOT CamelCase. They use `Iron_` prefix + `snake_case`.

**CORRECT APPROACH**: Skip extern func declarations for stdlib wrappers. Instead, declare object methods with EMPTY bodies, and have the auto-static codegen path (Task 2) directly emit the C function name using the naming convention: `Iron_<lowercase(type_name)>_<method_name>`.

When codegen sees `Math.sin(1.0)` on an auto-static method:
- Construct the C function name: `Iron_` + lowercase(type_name) + `_` + method_name
- `Iron_` + `math` + `_` + `sin` = `Iron_math_sin`
- Emit: `Iron_math_sin(1.0)`

The .iron wrapper files declare objects with empty-body methods. The auto-static codegen path ignores the method body and directly emits the C function call using the naming convention.

**A. Add auto-static dispatch in gen_exprs.c METHOD_CALL case (line ~525-558):**

Currently the METHOD_CALL case does:
1. Check if `mc->object` is an IRON_NODE_IDENT with `resolved_type->kind == IRON_TYPE_OBJECT` to get `type_name`
2. If found, emit `Iron_TypeName_method(&instance, args...)`
3. Otherwise, fallback to `object.method(args...)`

Add an auto-static check BEFORE the existing instance method dispatch. After the `type_name` resolution (line ~535), add a new check:

```c
case IRON_NODE_METHOD_CALL: {
    Iron_MethodCallExpr *mc = (Iron_MethodCallExpr *)node;

    /* AUTO-STATIC CHECK: If the receiver is a type name (IRON_SYM_TYPE),
     * emit a direct C function call without self pointer.
     * Convention: Iron_<lowercase_type>_<method>(args...)
     * This matches the stdlib C API naming: Iron_math_sin, Iron_io_read_file, etc. */
    if (mc->object->kind == IRON_NODE_IDENT) {
        Iron_Ident *obj_id = (Iron_Ident *)mc->object;
        if (obj_id->resolved_sym &&
            obj_id->resolved_sym->sym_kind == IRON_SYM_TYPE) {
            /* Auto-static: emit Iron_<lower(type)>_<method>(args...) */
            const char *tname = obj_id->name;  /* e.g. "Math" */

            /* Build lowercase version of type name */
            size_t tlen = strlen(tname);
            char *lower = (char *)iron_arena_alloc(ctx->arena, tlen + 1);
            for (size_t k = 0; k < tlen; k++) {
                lower[k] = (char)tolower((unsigned char)tname[k]);
            }
            lower[tlen] = '\0';

            /* Emit: Iron_<lower>_<method>(args...) */
            iron_strbuf_appendf(sb, "Iron_%s_%s(", lower, mc->method);
            for (int i = 0; i < mc->arg_count; i++) {
                if (i > 0) iron_strbuf_appendf(sb, ", ");
                emit_expr(sb, mc->args[i], ctx);
            }
            iron_strbuf_appendf(sb, ")");
            break;
        }
    }

    /* ... existing instance method dispatch code below (unchanged) ... */
```

This code:
- Checks if the receiver ident's resolved_sym is IRON_SYM_TYPE
- Constructs the C function name as `Iron_` + lowercase(typename) + `_` + methodname
- Emits a function call WITHOUT self pointer
- Math.sin(1.0) -> Iron_math_sin(1.0) -- correct!
- IO.read_file("test.txt") -> Iron_io_read_file(...) -- correct!
