<objective>
Implement spawn/await end-to-end: make spawn return a handle, await block and return the spawned task's result value, add the iron_future_await runtime function, add compiler warning for un-awaited spawn handles, and create integration tests.

Purpose: Iron's spawn currently fires-and-forgets with no way to retrieve results. This makes await non-functional and benchmark timing unfair vs C's pthread_join. This plan wires spawn -> handle -> await through the full compiler pipeline and runtime.
Output: Working `val h = spawn("name") { return value }; val r = await h` end-to-end, plus integration tests proving correctness.

Add to `src/runtime/iron_threads.c` (after `Iron_handle_create`):
```c
Iron_Handle *iron_handle_create_self_ref(void (*fn)(void *)) {
    Iron_Handle *h = (Iron_Handle *)malloc(sizeof(Iron_Handle));
    if (!h) return NULL;

    h->done      = false;
    h->result    = NULL;
    h->panic_msg = NULL;
    IRON_MUTEX_INIT(h->lock);
    IRON_COND_INIT(h->cond);

    HandleWrapper *w = (HandleWrapper *)malloc(sizeof(HandleWrapper));
    if (!w) {
        free(h);
        return NULL;
    }
    w->handle = h;
    w->fn     = fn;
    w->arg    = h;  /* <-- self-referential: pass handle as arg */

    IRON_THREAD_CREATE(h->thread, handle_thread_fn, w);
    return h;
}
```
