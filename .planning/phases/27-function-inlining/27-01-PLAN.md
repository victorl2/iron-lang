```c
static int count_func_instructions(IronLIR_Func *fn) {
    int count = 0;
    for (int bi = 0; bi < fn->block_count; bi++)
        count += fn->blocks[bi]->instr_count;
    return count;
}
```

**Step 2: Add helper — func_has_self_call:**
Check if any CALL in the function body calls itself (by name comparison). This is the recursion guard (Pitfall 7).

**Step 3: Add helper — is_locally_pure:**
Replicate Phase 1 logic from `compute_func_purity` but into a local `{ char *key; bool value; } *local_purity = NULL` map. The function scans all functions for leaf-only purity: no CALLs, and all instructions are either `iron_lir_instr_is_pure(kind)` or structural (RETURN, JUMP, ALLOCA, STORE). Do NOT write to `info->func_purity` (Pitfall 4). Only Phase 1 (leaf-only, no fixpoint) is needed for v1 of this pass.

**Step 4: Build inline candidate set** — `{ char *key; IronLIR_Func *value; } *candidates = NULL`:
For each function in module:
- Skip if `is_extern` or `block_count == 0`
- Skip if `count_func_instructions(fn) > 20`
- Skip if NOT in local purity map
- Skip if `func_has_self_call(fn)`
- Otherwise: `shput(candidates, fn->name, fn)` — mark as inline candidate

**Step 5: Main inlining loop** — For each function `fn` in module (the CALLER):
```
int orig_block_count = fn->block_count;  // Snapshot! (Pitfall: stb_ds realloc on arrput)
for (int bi = 0; bi < orig_block_count; bi++) {
    IronLIR_Block *blk = fn->blocks[bi];
    // Snapshot instruction list to avoid mutation-during-iteration
    int orig_instr_count = blk->instr_count;
    for (int ii = 0; ii < orig_instr_count; ii++) {
        IronLIR_Instr *instr = blk->instrs[ii];
        if (instr->kind != IRON_LIR_CALL) continue;

        // Resolve callee name from CALL instruction
        const char *callee_name = NULL;
        if (instr->call.func_decl) {
            callee_name = instr->call.func_decl->name;
        } else if (instr->call.func_ptr != IRON_LIR_VALUE_INVALID) {
            IronLIR_ValueId fptr = instr->call.func_ptr;
            if (fptr < (IronLIR_ValueId)arrlen(fn->value_table) &&
                fn->value_table[fptr] &&
                fn->value_table[fptr]->kind == IRON_LIR_FUNC_REF) {
                callee_name = fn->value_table[fptr]->func_ref.func_name;
            }
        }
        if (!callee_name) continue;

        // Check if callee is an inline candidate
        ptrdiff_t ci = shgeti(candidates, callee_name);
        if (ci < 0) continue;
        IronLIR_Func *callee = candidates[ci].value;

        // Don't inline a function into itself
        if (strcmp(fn->name, callee_name) == 0) continue;

        // ===== INLINE THIS CALL SITE =====
        inline_call_site(fn, blk, ii, instr, callee);
        // After inlining, blk's instructions have been modified.
        // Break out of the instruction loop for this block — we've split it.
        break;  // Process remaining original blocks; don't re-enter this block's instr loop
    }
}
```

**Step 6: Implement `inline_call_site(fn, call_block, call_idx, call_instr, callee)`:**

This is the most critical function. Algorithm:

**6a. Create value ID remap and block ID remap:**
```c
ValueReplEntry *id_remap = NULL;
BlockIdRemap *block_remap = NULL;
```

**6b. Clone callee blocks into caller:**
For each block in `callee->blocks[0..callee->block_count]`:
- Allocate new block via `ARENA_ALLOC(fn->arena, IronLIR_Block)` + `memset(0)`.
- Assign `new_blk->id = fn->next_block_id++`. Record `hmput(block_remap, callee_blk->id, new_blk->id)`.
- Set `new_blk->label` to a descriptive string (e.g., `"inline_entry"`, `"inline_body"`, etc. — use callee block label if available).
- For each instruction in callee block:
  - `IronLIR_Instr *cloned = ARENA_ALLOC(fn->arena, IronLIR_Instr); memcpy(cloned, src, sizeof(*src));`
  - If `src->id != IRON_LIR_VALUE_INVALID`: assign fresh ID via `cloned->id = fn->next_value_id++`, grow value_table (exact `make_alloca_instr` pattern), register `fn->value_table[cloned->id] = cloned`, add `hmput(id_remap, src->id, cloned->id)`.
  - If `src->id == IRON_LIR_VALUE_INVALID`: leave `cloned->id = IRON_LIR_VALUE_INVALID`.
  - If the instruction is a CALL and `src->call.args` is not NULL: deep-copy the args array since `memcpy` only copies the pointer. Allocate a fresh stb_ds array of IronLIR_ValueId, copy elements.
  - `arrput(new_blk->instrs, cloned); new_blk->instr_count++;`
- Append cloned block: `arrput(fn->blocks, new_blk); fn->block_count++;`

**6c. Apply value ID remap to all cloned instructions:**
For each cloned block, for each instruction: `apply_replacements(cloned_instr, id_remap)`.

**6d. Apply block ID remap to terminators in cloned blocks:**
For each cloned block, check the last instruction (terminator):
- JUMP: `hmgeti(block_remap, instr->jump.target)` -> if found, replace `instr->jump.target = block_remap[idx].value`.
- BRANCH: remap `instr->branch.then_block` and `instr->branch.else_block`.
- SWITCH: remap `instr->switch_instr.default_block` and each `instr->switch_instr.case_blocks[i]`.

**6e. Build parameter-to-argument substitution:**
After phi_eliminate, callee parameters are accessed via ALLOCA+STORE+LOAD chains. The entry block of the callee starts with ALLOCAs for each parameter (one per param). Map those cloned ALLOCA IDs (already remapped) to the CALL arguments. Approach:
- Walk the first `callee->param_count` instructions of the cloned entry block. These should be ALLOCAs (from phi_eliminate). For each param ALLOCA at index `p`:
  - Get the remapped ALLOCA ID from the cloned instruction.
  - The next instruction should be a STORE to this ALLOCA from a param value. We need to find the original param value IDs.
  
Actually, the simpler approach: after phi_eliminate, the callee's param values are accessed through ALLOCA+STORE+LOAD. The ALLOCA stores a param value. The copy-propagation in the fixpoint loop (which runs AFTER inlining) will eliminate these ALLOCA+STORE+LOAD chains automatically. So the param-to-arg mapping is:
- For each parameter `p` (0..param_count-1): find the STORE instruction in the cloned entry block that writes the initial param value to the param's ALLOCA. Replace that STORE's value with `call_instr->call.args[p]`.

Alternative simpler approach: Insert STORE instructions at the start of the cloned entry block that write each call argument into the corresponding param ALLOCA. The existing STOREs from the callee's entry (which store param values) need to be replaced. Since after phi_eliminate the callee entry block starts with: ALLOCA_0 (param 0), ALLOCA_1 (param 1), ..., STORE param_val_0 -> ALLOCA_0, STORE param_val_1 -> ALLOCA_1, ... the param_val IDs were callee-space IDs that no longer exist in the caller. Replace them:
- For each STORE in the cloned entry block whose `.store.ptr` matches a param ALLOCA's remapped ID: set `.store.value = call_instr->call.args[p]`.

If this pattern does not hold exactly (phi_eliminate may not produce this exact layout), use an alternative: prepend new STORE instructions to the cloned entry block that write `call_instr->call.args[p]` to the param ALLOCA. This is safe because copy-prop will eliminate the redundancy.

**6f. Create merge block with result ALLOCA:**
- Create merge block: `IronLIR_Block *merge = ARENA_ALLOC(fn->arena, IronLIR_Block); memset(merge, 0, sizeof(*merge)); merge->id = fn->next_block_id++; merge->label = "inline_merge";`
- If callee returns a value (return_type is not void): create result ALLOCA via `make_alloca_instr(fn, callee->return_type, call_instr->span)`. Insert it at the top of the merge block.
- Create a LOAD from the result ALLOCA (if non-void return). This LOAD's ID becomes the replacement for the original CALL's result ID.
- Append merge block: `arrput(fn->blocks, merge); fn->block_count++;`

**6g. Replace RETURN in cloned blocks with STORE + JUMP to merge:**
For each cloned block, if the last instruction is RETURN:
- If non-void: create STORE instruction (`make_store_instr(fn, result_alloca_id, remapped_ret_value, span)`) — but note `ret.value` was already remapped by `apply_replacements` in step 6c. Insert STORE before the RETURN position.
- Replace RETURN with a JUMP to the merge block: allocate new JUMP instruction with `jump.target = merge->id`.

**6h. Split caller block at the CALL site:**
The caller block (`call_block`) contains instructions before and after the CALL. Split it:
- Instructions `[0..call_idx-1]` stay in `call_block`.
- Instructions `[call_idx+1..orig_instr_count-1]` move to a new continuation block.
- The CALL instruction itself is removed.
- `call_block`'s terminator becomes a JUMP to the cloned entry block (first cloned block).
- The merge block's terminator becomes a JUMP to the continuation block.
- If the CALL produced a result (call_instr->id != INVALID): add `hmput(id_remap, call_instr->id, load_result_id)` and call `apply_replacements` on all instructions in the continuation block to propagate the result.

Create the continuation block:
```c
IronLIR_Block *cont = ARENA_ALLOC(fn->arena, IronLIR_Block);
memset(cont, 0, sizeof(*cont));
cont->id = fn->next_block_id++;
cont->label = "inline_cont";
// Move instructions after CALL to continuation
for (int k = call_idx + 1; k < call_block->instr_count; k++) {
    arrput(cont->instrs, call_block->instrs[k]);
    cont->instr_count++;
}
// Truncate call_block to instructions before CALL
call_block->instr_count = call_idx;
arrsetlen(call_block->instrs, call_idx);
```

Add JUMP from call_block to cloned entry:

```c
jump_to_entry->kind = IRON_LIR_JUMP;
jump_to_entry->id = IRON_LIR_VALUE_INVALID;
```

```c
jump_to_cont->id = IRON_LIR_VALUE_INVALID;
```

**CRITICAL PITFALLS to guard against:**
1. **Value ID collision (Pitfall 1):** EVERY cloned instruction with `id != IRON_LIR_VALUE_INVALID` must get `fn->next_value_id++` and be registered in value_table. Use the exact `make_alloca_instr` pattern.
2. **Block ID collision (Pitfall 2):** ALL cloned blocks get `fn->next_block_id++`. ALL terminator targets remapped via block_remap.
3. **RETURN handling (Pitfall 3):** Replace with STORE + JUMP, never leave RETURN in cloned code.
4. **stb_ds iteration safety:** Snapshot block_count BEFORE the loop. `arrput` may reallocate — never hold stale block pointers across arrput. Re-fetch `fn->blocks[bi]` after any arrput.
5. **Deep-copy CALL args:** `memcpy` of a CALL instruction copies only the args pointer. The cloned CALL must have its own args array.
6. **Purity map (Pitfall 4):** Use LOCAL purity map, not `info->func_purity`.
7. **Recursion guard (Pitfall 7):** Exclude functions with self-calls from candidate set.

**Note on BRANCH-like instructions in callee bodies:** `find_root` has a BRANCH (the while loop). After block ID remapping, the BRANCH targets in cloned blocks will point to other cloned blocks — which is correct because those blocks are now in the caller's block array.
