**Test 1: audit_pfor_variable_range.iron**
Exercises pfor with a variable range expression (not an integer literal). All existing pfor tests use literal ranges like `for i in 50 parallel`. This test uses a variable.
```iron
func main() {
```

**Test 2: audit_struct_chain_passing.iron**
Exercises struct passed by value through a chain of 3+ functions: create -> transform -> consume. This stresses the CALL argument emission path for struct-typed parameters and return values across multiple function boundaries.
```iron
object Vec2 {
  var x: Int
  var y: Int
}

func make_vec(x: Int, y: Int) -> Vec2 {
  return Vec2(x, y)
}

func scale(v: Vec2, factor: Int) -> Vec2 {
  return Vec2(v.x * factor, v.y * factor)
}

func add(a: Vec2, b: Vec2) -> Vec2 {
  return Vec2(a.x + b.x, a.y + b.y)
}

func dot(a: Vec2, b: Vec2) -> Int {
  return a.x * b.x + a.y * b.y
}

func main() {
  val v1 = make_vec(3, 4)
  val v2 = scale(v1, 2)
  val v3 = add(v1, v2)
  println("v1: ({v1.x}, {v1.y})")
  println("v2: ({v2.x}, {v2.y})")
  println("v3: ({v3.x}, {v3.y})")
  println("dot: {dot(v1, v2)}")
  println("chain: {dot(add(v1, v2), scale(v3, 3))}")
}
```
Expected output (compute manually):
- v1 = (3, 4)
- v2 = scale(v1, 2) = (6, 8)
- v3 = add(v1, v2) = (9, 12)
- dot(v1, v2) = 3*6 + 4*8 = 18 + 32 = 50
- add(v1, v2) = (9, 12), scale(v3, 3) = (27, 36), dot = 9*27 + 12*36 = 243 + 432 = 675
```
v1: (3, 4)
v2: (6, 8)
v3: (9, 12)
dot: 50
chain: 675
```

**Test 3: audit_struct_method_mutation.iron**
Exercises struct field mutation inside object methods. The `self` parameter in Iron methods is passed by value, so mutation inside the method should affect only the local copy. This tests the method call emission path.
```iron
object Counter {
  var count: Int
  var name: String
}

func Counter.increment(amount: Int) -> Counter {
  return Counter(self.count + amount, self.name)
}

func Counter.display() {
  println("{self.name}: {self.count}")
}

func main() {
  val c1 = Counter(0, "alpha")
  val c2 = c1.increment(5)
  val c3 = c2.increment(3)
  c1.display()
  c2.display()
  c3.display()
}
```
Expected output:
```
alpha: 0
alpha: 5
alpha: 8
```

**Test 4: audit_for_array_of_structs.iron**
Exercises for-in loop over an array of structs with field access inside the loop body. The existing hir_edge_array_of_objects test uses a while loop. This test uses for-in, exercising the GET_INDEX + GET_FIELD chain on object arrays through the for-loop lowering path.
```iron
object Item {
  var name: String
  var value: Int
}

func main() {
  var items = [Item("apple", 3), Item("banana", 5), Item("cherry", 2)]
  var total = 0
  for item in items {
    println("{item.name}: {item.value}")
    total = total + item.value
  }
  println("total: {total}")
}
```
Expected output:
```
apple: 3
banana: 5
cherry: 2
total: 10
```

**Test 5: audit_defer_in_for_body.iron**
Exercises defer inside a for-loop body. Each iteration should execute its deferred statement at the end of that iteration's scope. This tests the push_defer_scope/emit_scope_defers interaction with for-loop block boundaries.
```iron
func main() {
  var log = ""
  for i in 3 {
    defer {
      println("defer {i}")
    }
    println("start {i}")
  }
  println("done")
}
```
Expected output (defer executes at end of each iteration, reverse order within iteration):
```
start 0
defer 0
start 1
defer 1
start 2
defer 2
done
```

**Test 6: audit_pfor_boundary_range.iron**
Exercises pfor with small boundary range values (1 and 2) to verify the _total, _nthreads, _chunk_size calculations do not produce incorrect loop bounds for very small ranges.
```iron
func main() {
  for i in 1 parallel {
    var x = i + 1
  }
  println("range 1 done")

  for i in 2 parallel {
    var x = i + 1
  }
  println("range 2 done")

  for i in 3 parallel {
    var x = i + 1
  }
  println("range 3 done")
}
```
Expected output:
```
range 1 done
range 2 done
range 3 done
```
