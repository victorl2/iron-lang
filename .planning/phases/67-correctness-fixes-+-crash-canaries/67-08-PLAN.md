```iron
-- Regression: REG-02 crash-canary for IRON_HIR_STMT_IF with elif/else branches.
--
-- **Motivating Incident.** Phase 37-02's collect_mono_enums_node SIGSEGV in
-- src/hir/hir_to_lir.c was caused by a locally-shadowed Iron_IfStmt struct
-- guess that omitted elif_conds[] / elif_bodies[] fields between `body` and
-- `else_body`. The walker crashed deterministically on Linux x86_64 Release
-- for any program containing an if/elif/else chain, but macOS arm64 masked
-- the bug. Phase 66 added _Static_assert on the real Iron_IfStmt layout
-- plus -Werror=switch-enum; this canary is the runtime regression net so
-- any future walker regression for IRON_NODE_IF surfaces immediately in
-- both Debug and Release CI.
--
-- **Layout Diagram.** Iron_IfStmt carries {span, kind, condition, body,
-- elif_conds[], elif_bodies[], elif_count, else_body}. Any walker case
-- that fails to recurse into elif_conds[] or elif_bodies[] silently drops
-- sub-expressions from the lowered LIR.
--
-- **Fix Summary.** Phase 67-08 added this fixture as a REG-02 crash canary.
-- Complements the existing hir_to_lir_elif_mono_walker.iron (Phase 59 fix
-- fixture) — that one is the reference, this is the catalogue entry.
-- Compiles and runs cleanly in both Debug and Release.
--
-- **Severity.** H — the Phase 37-02 SIGSEGV was a deterministic Linux
-- Release crash that hid for months because per-node-kind canaries did
-- not exist.

func bucket(n: Int) -> Int {
    if n < -10 {
        return 0
    } elif n < 0 {
        return 1
    } elif n == 0 {
        return 2
    } elif n < 10 {
        return 3
    } elif n < 100 {
        return 4
    } else {
        return 5
    }
}

func main() {
    var total = 0
    for i in range(-20, 20, 5) {
        total = total + bucket(i)
    }
    println("total: {total}")
}
```
