Replace the stub `src/ir/print.c` with the full implementation of `iron_ir_print()`.

**Function signature:** `char *iron_ir_print(const IronIR_Module *module, bool show_annotations)`
- Returns a heap-allocated string (malloc'd via strbuf). Caller must free with `free()`.
- Uses Iron_StrBuf internally to build the output.
- Needs a temporary Iron_Arena for iron_type_to_string calls (create/free within the function).

**Output format (locked by CONTEXT.md):**

1. **Module header:**
```
; Module: <name>
```

2. **Type declarations section** (iterate module->type_decls):
```
type @Iron_Player = { x: Int, y: Float }
type @Iron_Direction = enum { North = 0, South = 1 }
interface @Iron_Drawable = { draw: func(@Iron_Drawable_ref) -> Void }
```
For type_decls, print kind-appropriate format. For now, print: `type @<name> = <iron_type_to_string(type)>`

3. **Extern declarations** (iterate module->extern_decls):
```
extern @<c_name>(<param_name>: <type>, ...) -> <return_type>
```

4. **Blank line separator, then function bodies** (iterate module->funcs):
```
func @<name>(<param_name>: <type>, ...) -> <return_type> {
```
For each block in func->blocks:
```
<label>:
  <instructions>
```

5. **Instruction printing** -- switch on `instr->kind` with a case for EVERY variant. Format per kind:

- **Constants:**
  - `  %<id> = const_int <value> : <type>`
  - `  %<id> = const_float <value> : <type>`
  - `  %<id> = const_bool <true|false> : <type>`
  - `  %<id> = const_string "<value>" : <type>`
  - `  %<id> = const_null : <type>`

- **Binary ops** (ADD, SUB, MUL, DIV, MOD, EQ, NEQ, LT, LTE, GT, GTE, AND, OR):
  - `  %<id> = <op_name> %<left>, %<right> : <type>`
  - op_name is lowercase of the enum name without IRON_IR_ prefix (e.g. "add", "sub", "eq", "neq")

- **Unary ops** (NEG, NOT):
  - `  %<id> = <op_name> %<operand> : <type>`

- **Memory:**
  - `  %<id> = alloca <alloc_type>` (with `; name_hint: "<hint>"` annotation if show_annotations and name_hint != NULL)
  - `  %<id> = load %<ptr> : <type>`
  - `  store %<ptr>, %<value>` (no %id since void)

- **Field/Index:**
  - `  %<id> = get_field %<object>.<field> : <type>`
  - `  set_field %<object>.<field>, %<value>`
  - `  %<id> = get_index %<array>[%<index>] : <type>`
  - `  set_index %<array>[%<index>], %<value>`

- **Call:**
  - Direct: `  %<id> = call @<func_decl->name>(%<arg0>, %<arg1>, ...) : <type>`
  - Indirect: `  %<id> = call %<func_ptr>(%<arg0>, ...) : <type>`
  - Void call: `  call @<name>(...)` (no %id)

- **Control flow (terminators):**
  - `  jump <target_label>` (resolve block ID to label via func->blocks array)
  - `  branch %<cond>, <then_label>, <else_label>`
  - `  switch %<subject>, default: <default_label>, [<val0>: <label0>, <val1>: <label1>, ...]`
  - `  ret %<value>` or `  ret void`

- **Cast:** `  %<id> = cast %<value> : <target_type>`

- **Heap/RC/Free:**
  - `  %<id> = heap_alloc %<inner_val> : <type>` (with `; auto_free` and/or `; escapes` annotations)
  - `  %<id> = rc_alloc %<inner_val> : <type>`
  - `  free %<value>`

- **Construct/Array/Slice:**
  - `  %<id> = construct <type>(%<field0>, %<field1>, ...) : <type>`
  - `  %<id> = array_lit [%<elem0>, %<elem1>, ...] : <type>`
  - `  %<id> = slice %<array>[%<start>:%<end>] : <type>` (use "begin" for INVALID start, "end" for INVALID end)

- **Null checks:**
  - `  %<id> = is_null %<value> : Bool`
  - `  %<id> = is_not_null %<value> : Bool`

- **String interpolation:**
  - `  %<id> = interp_string [%<part0>, %<part1>, ...] : <type>`

- **Closures/Refs:**
  - `  %<id> = make_closure @<lifted_func_name>(%<cap0>, %<cap1>, ...) : <type>`
  - `  %<id> = func_ref @<func_name> : <type>`

- **Concurrency:**
  - `  %<id> = spawn @<lifted_func_name>, pool: %<pool_val>` (or `pool: default` if INVALID)
  - `  parallel_for <loop_var>, %<range>, chunk: @<chunk_func>, pool: %<pool_val>, captures: [%<cap0>, ...]`
  - `  %<id> = await %<handle> : <type>`

- **Phi:**
  - `  %<id> = phi [%<val0>: <pred_label0>, %<val1>: <pred_label1>, ...] : <type>`

6. **Annotations** (when show_annotations is true):
   - After heap_alloc: append `; auto_free` if auto_free flag, `; escapes` if escapes flag
   - After alloca: append `; name_hint: "<name>"` if name_hint is not NULL
   - After each block label: append `; preds: <pred_labels>` if preds array is non-empty
   - Source span comments are deferred to a future enhancement (not required for Phase 7)

7. **Helper function** to resolve IronIR_BlockId to label string:
```c
static const char *resolve_block_label(const IronIR_Func *fn, IronIR_BlockId id)
```
Walk fn->blocks to find the block with matching id, return its label. Return "<invalid>" if not found.

8. **Critical:** Include a `default: assert(false && "unhandled IronIR_InstrKind in printer");` in the switch to catch missing variants at test time (Pitfall 5 from research).

Replace the stub `src/ir/verify.c` with the full implementation of `iron_ir_verify()`.

**Function signature:** `bool iron_ir_verify(const IronIR_Module *module, Iron_DiagList *diags, Iron_Arena *arena)`
- Returns true if the module is structurally valid.
- Reports each violation via `iron_diag_emit(diags, arena, IRON_DIAG_ERROR, code, span, message, suggestion)`.
- Does NOT stop at the first error -- reports all violations found.

**Invariants to check (one per function in the module, for each block in the function):**

1. **IRON_ERR_IR_NO_ENTRY_BLOCK (304):** Every function must have at least one block (func->block_count >= 1).
   - Span: use iron_span_make with the module name and 0,0,0,0 since no block exists
   - Message: `"function '%s' has no basic blocks"` (func->name)
   - Suggestion: `"add at least an entry block"`

2. **IRON_ERR_IR_MISSING_TERMINATOR (300):** The last instruction in every block must be a terminator (JUMP, BRANCH, SWITCH, or RETURN). Check using `iron_ir_is_terminator(last_instr->kind)`.
   - Span: span of the last instruction in the block, or if block has zero instructions, the span of the first instruction in the function's entry block (or zero span if no instructions anywhere)
   - Message: `"basic block '%s' has no terminator"` (block->label)
   - Suggestion: `"add a return, jump, or branch as the last instruction"`

3. **IRON_ERR_IR_INSTR_AFTER_TERMINATOR (303):** No instruction may appear after a terminator in a block. Walk instructions; once a terminator is seen, any subsequent instruction is a violation.
   - Span: span of the offending instruction (the one AFTER the terminator)
   - Message: `"instruction after terminator in block '%s'"` (block->label)
   - Suggestion: `"move this instruction before the terminator or to a new block"`

4. **IRON_ERR_IR_INVALID_BRANCH_TARGET (301):** Every block ID referenced in JUMP/BRANCH/SWITCH terminators must correspond to a valid block in the same function. Build a temporary set of valid block IDs (walk func->blocks, collect all block->id values). Then check:
   - JUMP: target must be in valid set
   - BRANCH: then_block and else_block must be in valid set
   - SWITCH: default_block and all case_blocks must be in valid set
   - Span: span of the terminator instruction
   - Message: `"branch target block ID %u does not exist in function '%s'"` (the invalid block id, func->name)
   - Suggestion: `"check that the target block was added to the function"`

5. **IRON_ERR_IR_USE_BEFORE_DEF (302):** Every IronIR_ValueId used as an operand in an instruction must be defined (exist in func->value_table). For Phase 7, use simplified checking:
   - For each instruction, collect all operand ValueIds (from binop.left/right, unop.operand, load.ptr, store.ptr/value, branch.cond, call.args, etc.)
   - For each operand: check `operand != IRON_IR_VALUE_INVALID && (operand >= arrlen(fn->value_table) || fn->value_table[operand] == NULL)` -- if true, it's use-before-def
   - Span: span of the instruction that uses the undefined value
   - Message: `"use of undefined value %%u"` (the operand id)
   - Suggestion: `"ensure the value is defined before this instruction"`

6. **IRON_ERR_IR_RETURN_TYPE_MISMATCH (305):** RETURN instructions must match the function's declared return type.
   - If `func->return_type == NULL` (void function): return instruction must have `is_void == true`
   - If `func->return_type != NULL`: return instruction must have `is_void == false` and the return value's type (look up in value_table) must equal func->return_type (use iron_type_equals)
   - Span: span of the return instruction
   - Message: `"return type mismatch in function '%s'"` (func->name)
   - Suggestion: `"return value type must match function signature"`

**Helper functions:**
- `static void collect_operands(const IronIR_Instr *instr, IronIR_ValueId *out, int *count)` -- extract all ValueId operands from an instruction into an array. Handle each instruction kind. Skip IRON_IR_VALUE_INVALID entries.
- `static bool block_id_valid(const IronIR_Func *fn, IronIR_BlockId id)` -- check if block ID exists in fn->blocks.

**Return:** true only if diags->error_count has not increased from its initial value at the start of the call.
