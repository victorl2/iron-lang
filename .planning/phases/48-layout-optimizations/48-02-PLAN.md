<objective>
Implement automatic SoA/AoS layout selection based on field access patterns and common field factoring across implementors.

Purpose: When a loop accesses only a few fields of collection elements, SoA layout puts those fields in contiguous arrays for cache-optimal iteration. When all fields are accessed, AoS is better. Common fields shared across all implementors should be in a single array to avoid duplication. These are the core performance optimizations that make Iron's data-oriented design work.

Output: Extended layout_analysis with access pattern analysis, SoA struct generation in emit_c.c, common field shared arrays, two integration tests.
</objective>

**Extend layout_analysis.h** with:

```c
/* SoA/AoS layout selection */
#define IRON_SOA_THRESHOLD 0.5  /* SoA when accessed_fields / total_fields < this */

typedef enum {
    IRON_LAYOUT_AOS,  /* Array of Structs (default) */
    IRON_LAYOUT_SOA   /* Structure of Arrays */
} IronLayoutKind;

/* Per (collection_vid, type_name, loop_block_id) layout decision */
typedef struct {
    char *key;           /* "vid:type:block_id" */
    IronLayoutKind value;
} LayoutDecision;

/* Common field entry: field shared across all alive implementors */
typedef struct {
    const char *name;
    const char *c_type;  /* C type string for emission */
    int position;        /* field index in original struct */
} CommonField;

/* Analyze per-loop access patterns and determine SoA vs AoS per (collection, type, loop).
 * Must be called AFTER iron_layout_analyze() has populated field_used data.
 * Populates la->layout_decisions and la->common_fields. */
void iron_layout_select(LayoutAnalysis *la,
                        IronLIR_Module *module,
                        struct { IronLIR_ValueId key; const char *value; } *split_collection_ids,
                        Iron_IfaceRegistry *iface_reg);

/* Get layout kind for a specific (collection, type, loop). Returns IRON_LAYOUT_AOS if no data. */
IronLayoutKind iron_layout_get_kind(LayoutAnalysis *la,
                                     IronLIR_ValueId collection_vid,
                                     const char *type_name);

/* Get common fields for an interface. Returns stb_ds array (NULL if none).
 * Caller must NOT free the returned array. */
CommonField *iron_layout_get_common_fields(LayoutAnalysis *la,
                                            const char *iface_name);
```

**Implement in layout_analysis.c:**

1. `iron_layout_select()`:
   - For each function, for each split loop (detect using same pattern as emit_c.c: for_pre blocks referencing split collections), analyze the loop body:
     a. Count total fields per implementor type (from `impl->decl->field_count`)
     b. Count accessed fields in the loop body by scanning GET_FIELD instructions where the object traces to the loop variable (same tracing as iron_layout_analyze)
     c. Compute ratio = accessed_fields / total_fields
     d. If ratio < IRON_SOA_THRESHOLD: select SoA, else AoS
     e. Store decision keyed by `"vid:type_name"` (simplified: one decision per collection per type, the most restrictive loop wins — if ANY loop wants SoA, use SoA)

   Note: Per CONTEXT.md, analysis is per loop body per collection. But since we generate ONE struct per interface, we need a single decision per (collection_vid, type). Use the loop with the lowest access ratio to drive the decision (most SoA-friendly). If multiple loops access the same collection, the one accessing fewest fields wins.

   **Simplification for v1:** Since the SplitList struct is per-interface (not per-collection instance), use the lowest access ratio across ALL collections of that interface across ALL loops. This means if ANY usage is SoA-favorable, the entire interface uses SoA.

   Final key format: `"iface_mangled:type_name"` -> IronLayoutKind

2. `iron_layout_get_kind()`: Look up `"iface_mangled:type_name"` in layout_decisions. Return AOS if not found.

3. Common field detection in `iron_layout_select()`:
   - For each interface in iface_reg:
     a. Get all alive implementors
     b. For position 0, 1, 2, ...: check if ALL alive implementors have a field at that position with the same name and compatible type annotation
     c. Compatible type: both have the same `type_ann->name` string (e.g., both "Int" or both "Float")
     d. If a position has a common field, add to the CommonField array for this interface
   - Store in `la->common_fields` keyed by interface name

4. `iron_layout_get_common_fields()`: Look up interface name in common_fields map. Return NULL if not found.

Update `iron_layout_free()` to also free `layout_decisions` and `common_fields`.

```iron
-- layout_soa_select.iron: SoA layout selected for partial-field access
-- Tests Phase 48: automatic SoA when loop accesses < 50% of fields

interface Entity {
  func get_x() -> Int
}

object Particle impl Entity {
  var x: Int
  var y: Int
  var z: Int
  var mass: Int
}

object Bullet impl Entity {
  var x: Int
  var y: Int
  var z: Int
  var speed: Int
}

func Particle.get_x() -> Int {
  return self.x
}

func Bullet.get_x() -> Int {
  return self.x
}

func main() {
  val entities = [Particle(10, 20, 30, 100), Bullet(5, 15, 25, 50), Particle(3, 6, 9, 200)]
  var sum_x = 0
  for e in entities {
    -- Only accesses x (1 of 4 fields = 25% < 50% threshold -> SoA)
    sum_x = sum_x + e.get_x()
  }
  -- 10 + 5 + 3 = 18
  println("{sum_x}")
  println("done")
}
```
