Build field access analysis infrastructure and implement dead field elimination for split collection storage structs.

Purpose: When a split collection only uses some fields through interface operations, the storage struct should exclude unused fields to reduce memory footprint and improve cache utilization. This is the foundation for all subsequent layout optimizations (SoA, common field factoring) since they all depend on knowing which fields are accessed per collection.

Output: layout_analysis.h/c with field usage tracking, modified emit_c.c that generates reduced storage structs with direct field access, integration test proving dead fields are excluded.

Create `src/lir/layout_analysis.c`:

The analysis scans all functions for GET_FIELD/SET_FIELD on values derived from split collection iteration. For each such field access, it marks the field as used for that collection.

**Algorithm:**
1. For each function in module:
   a. Build a per-function map: `alloca_to_split` mapping ALLOCA ValueIds to split collection ValueIds. Populated by scanning for the pattern: GET_INDEX(split_vid, idx) -> STORE(alloca, result) which is the standard for-loop pattern.
   b. Also build `value_to_split` mapping GET_INDEX result ValueIds to their source split collection ValueId.
   c. For each GET_FIELD/SET_FIELD instruction:
      - If `field.object` is directly a GET_INDEX result from a split collection (check value_to_split) -> mark field used for that collection
      - If `field.object` is a LOAD of an ALLOCA that traces to a split collection (check alloca_to_split via: find the LOAD's ptr, then check alloca_to_split) -> mark field used
2. To mark a field: format key as `"%u:%s"` (collection_vid:field_name) — note we use just collection_vid + field_name (not per-type, since the field access is through the interface and applies to all implementors that have that field).

**Query function** `iron_layout_is_field_used`:
- Format key as `"%u:%s"` (collection_vid:field_name)
- If found in map: return the stored value
- If NOT found: return false (field is NOT used = dead field)
- Special case: if `la->field_used` is NULL (no analysis ran): return true (conservative)

Add `src/lir/layout_analysis.c` to the sources list in `src/CMakeLists.txt` next to `src/lir/emit_c.c`.

**Step 2: Module-level prescan for split collections**

Create a new static function `prescan_split_collections(EmitCtx *ctx)` that:
1. Iterates ALL functions in `ctx->module`
2. For each function, iterates all blocks and instructions
3. Finds ARRAY_LIT instructions with `elem_type->kind == IRON_TYPE_INTERFACE`
4. Records them in `ctx->split_collection_ids` (same logic as the existing per-function prescan at line ~4322-4337)
5. After scanning all functions, calls `iron_layout_analyze(&ctx->layout, ctx->module, ctx->split_collection_ids, ctx->iface_reg)`

Call `prescan_split_collections(ctx)` in `iron_lir_emit_c()` BEFORE `emit_type_decls(ctx)`.

**Step 3: Emit reduced storage structs in SplitList generation**

In the SplitList struct generation (around line 4896-4922 where per-type sub-arrays are emitted), for each alive implementor:

a. Count used fields by iterating `impl->decl->fields[i]` and calling `iron_layout_is_field_used(&ctx->layout, /*collection_vid*/, field->name)`. The collection_vid is tricky here since we're generating a GENERIC SplitList for the interface, not per-collection. 

**Design decision:** Since different collections of the same interface may use different fields, but the SplitList struct is shared per interface, we must use the UNION of all used fields across all collections of that interface. Build a temporary set: for each split_collection_ids entry that maps to this interface, for each field, if ANY collection uses it, include it.

b. If all fields are used (common case): emit `Iron_<Type> *<lower>_items;` as before (no change).

c. If some fields are excluded: emit a reduced storage typedef BEFORE the SplitList struct:
```c
typedef struct {
    <c_type> <field_name>;
    /* only used fields */
} Iron_<Type>_Stor;
```
Then in the SplitList, emit `Iron_<Type>_Stor *<lower>_items;` instead of `Iron_<Type> *<lower>_items;`.

d. Record which types have reduced storage in a per-interface stb_ds map so the push function and loop emit can check.

**Step 4: Update push functions for reduced storage**

In the push function generation (around line 4940-4960), if the type has reduced storage:
Instead of `_sl-><lower>_items[_sl-><lower>_count] = _val;` (full struct copy), emit per-field copies for only the used fields:
```c
_sl->circle_items[_sl->circle_count].radius = _val.radius;
// debug_name NOT copied — dead field
```
The function still accepts the FULL struct `Iron_Circle _val` as parameter (since the caller pushes a full concrete object), but only copies the used fields into storage.

**Step 5: Update per-type loop emission for direct field access**

In the per-type loop emission (around line 4506-4539), when emitting for a type with reduced storage:
- Do NOT emit the tagged union wrapper line (`Iron_Shape _vN = Iron_Shape_from_Circle(sl.circle_items[_sp_i]);`)
- Instead, set `ctx->in_split_loop = true` and fill `ctx->split_loop_ctx` with the current loop's details
- After emitting body instructions, set `ctx->in_split_loop = false`

When reduced storage is NOT in use (all fields present): keep existing behavior (tagged union wrapper).

```iron
-- layout_dead_field.iron: Dead field elimination in split collections
-- Tests Phase 48: unused fields excluded from collection storage

interface Shape {
  func area() -> Int
}

object Circle impl Shape {
  var radius: Int
  var debug_name: Int
}

object Square impl Shape {
  var side: Int
  var debug_tag: Int
}

func Circle.area() -> Int {
  return self.radius * self.radius * 3
}

func Square.area() -> Int {
  return self.side * self.side
}

func main() {
  -- debug_name and debug_tag are never accessed through the collection
  val shapes = [Circle(5, 99), Square(4, 88), Circle(3, 77)]
  var total = 0
  for s in shapes {
    total = total + s.area()
  }
  -- Circle(5)=75, Square(4)=16, Circle(3)=27 = 118
  println("{total}")

  -- Direct concrete access still works
  val c = Circle(10, 42)
  println("{c.debug_name}")

  println("done")
}
```
