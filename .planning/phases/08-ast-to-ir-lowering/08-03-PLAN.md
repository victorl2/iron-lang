**lower_module_decls(ctx) — Pass 1 implementation (MOD-01, MOD-02, MOD-03):**

Iterate `ctx->program->decls` and register each declaration in the IrModule:

- **IRON_NODE_INTERFACE_DECL** (register FIRST for vtable ordering — MOD-01):
  - Cast to `Iron_InterfaceDecl*`
  - `iron_ir_module_add_type_decl(ctx->module, IRON_IR_TYPE_INTERFACE, iface->name, iface->type)`
  - Note: interfaces must be registered before objects that implement them (vtable dependency)

- **IRON_NODE_OBJECT_DECL** (MOD-01):
  - Cast to `Iron_ObjectDecl*`
  - `iron_ir_module_add_type_decl(ctx->module, IRON_IR_TYPE_OBJECT, obj->name, obj->type)`
  - For each method: create IrFunc signature (no body yet) via `iron_ir_func_create`
  - Handle generic objects: if obj has type params, iterate all known instantiations and register each via mono_registry

- **IRON_NODE_ENUM_DECL** (MOD-01):
  - Cast to `Iron_EnumDecl*`
  - `iron_ir_module_add_type_decl(ctx->module, IRON_IR_TYPE_ENUM, enum_decl->name, enum_decl->type)`

- **IRON_NODE_FUNC_DECL** (register signatures):
  - Cast to `Iron_FuncDecl*`
  - Build IronIR_Param array from fd->params
  - `iron_ir_func_create(ctx->module, fd->name, params, fd->param_count, fd->return_type)`
  - arrfree params
  - If function is marked extern: set `func->is_extern = true` and `func->extern_c_name = fd->c_name` (or equivalent field from ast.h)

- **IRON_NODE_IMPORT_DECL**:
  - Handle extern functions from imported modules. Check ast.h for how imports expose extern declarations. For each extern function in the import, call `iron_ir_module_add_extern()`.

- **Monomorphization (MOD-03):**
  - Walk all declarations looking for generic type instantiations (similar to how codegen's ensure_monomorphized_type works)
  - For each concrete generic use (e.g., List[Int]), compute mangled name via `mangle_generic()` (from codegen.h)
  - Check `shgeti(ctx->module->mono_registry, mangled_name)` — if not found, `shput(ctx->module->mono_registry, mangled_name, true)` and register the instantiation
  - Note: The actual monomorphized code emission is Phase 9's job. Pass 1 only registers the instantiation in the registry so Pass 2 knows about all concrete types.

Two passes over declarations: first register interfaces, then objects/enums/funcs. This ensures vtable types are available when objects reference them.

**lower_lift_pending(ctx) — Post-pass implementation (CONC-01, CONC-02, CONC-03):**

After all function bodies are lowered, process pending lifts:

```c
void lower_lift_pending(IronIR_LowerCtx *ctx) {
    for (int i = 0; i < (int)arrlen(ctx->pending_lifts); i++) {
        LiftPending *lp = &ctx->pending_lifts[i];
        switch (lp->kind) {
            case LIFT_LAMBDA:  lift_lambda(ctx, lp);  break;
            case LIFT_SPAWN:   lift_spawn(ctx, lp);   break;
            case LIFT_PARALLEL_FOR: lift_pfor(ctx, lp); break;
        }
    }
}
```

**lift_lambda (CONC-01):**
- Get the lambda AST node from `lp->ast_node`
- Run `collect_captures(body, params, param_count)` to find captured variables
- Create a new IronIR_Func with name = lp->lifted_name
- If captures exist:
  - First param is `env` (void* pointing to environment struct)
  - Remaining params are the lambda's declared params
  - In the function body: emit loads from env struct fields for each captured variable
  - Lower the lambda body with these capture-loaded values in val_binding_map
- If no captures:
  - Params are just the lambda's declared params
  - Lower body directly
- Insert the lifted IrFunc into ctx->module->funcs BEFORE the enclosing function (to satisfy forward declaration ordering — per research Pitfall 7)
- Find the enclosing function's index in module->funcs and insert before it using stb_ds arrsetlen + memmove, or simply append (the C emitter in Phase 9 will handle prototypes)

**lift_spawn (CONC-02):**
- Similar to lift_lambda but for spawn bodies
- The spawned function takes captures as parameters (no env struct needed since pool_submit can pass args)
- Create IrFunc with lp->lifted_name, lower the spawn body as the function body
- Register in module

**lift_pfor (CONC-03):**
- The parallel-for chunk function takes (start, end, captures...)
- Run collect_captures on the pfor body
- Create IrFunc with params: loop_var range params + captured variables
- Lower the chunk body
- Register in module
- The IR_PARALLEL_FOR instruction at the call site references this chunk function by name and passes captures
