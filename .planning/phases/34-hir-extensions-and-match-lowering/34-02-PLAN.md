**Step 3: Add `emit_nested_pattern_bindings` helper function.**

Create a static helper function (before the `lower_stmt` function) that recursively handles nested patterns:

```c
static void emit_nested_pattern_bindings(HIR_to_LIR_Ctx *ctx,
                                          IronLIR_ValueId parent_val,
                                          Iron_Type *parent_type,
                                          IronHIR_Expr *pattern,
                                          Iron_Span span) {
    /* pattern->kind == IRON_HIR_EXPR_PATTERN */
    /* 1. Extract tag from parent_val */
    /* 2. Compare tag to pattern's variant_index (emit BRANCH) */
    /* 3. In the matched block, extract payload fields */
    /* 4. For each non-wildcard binding: hoist ALLOCA, store payload value */
    /* 5. For each nested_pattern that is non-NULL, recurse */

    /* Get the enum decl from parent_type */
    Iron_EnumDecl *ed = parent_type->enu.decl;
    const char *vname = pattern->pattern.variant_name;

    /* Find variant index */
    int vidx = -1;
    for (int i = 0; i < ed->variant_count; i++) {
        Iron_EnumVariant *ev = (Iron_EnumVariant *)ed->variants[i];
        if (strcmp(ev->name, vname) == 0) { vidx = i; break; }
    }

    Iron_Type *int_type = iron_type_make_int(ctx->lir_arena);
    IronLIR_ValueId inner_tag = iron_lir_get_field(ctx->current_func, ctx->current_block,
                                                     parent_val, "tag", int_type, span)->id;

    /* For nested patterns, we don't need a full switch — just a conditional branch
       (the exhaustiveness check was done by the type checker in Phase 33).
       But we still need to guard: if tag != expected, skip this arm's bindings.
       Since we're already inside a matched outer arm, the inner tag check is
       a refinement. Use a branch: if inner_tag == vidx, go to bind_block, else skip. */

    /* Actually, for the generated C, inline tag checks are the decision from CONTEXT.md.
       Just emit the bindings directly (no branch needed at LIR level — the SWITCH +
       GET_FIELD path will produce if-checks in C). For simplicity and correctness,
       extract bindings conditionally. In the simplest case, since exhaustiveness is
       already verified, we can emit the bindings unconditionally within the arm. */

    /* Extract each binding */
    Iron_EnumVariant *ev = (Iron_EnumVariant *)ed->variants[vidx];
    for (int b = 0; b < pattern->pattern.binding_count; b++) {
        const char *bname = pattern->pattern.binding_names[b];
        Iron_Type *payload_type = ev->payload_types[b];

        char field_path[256];
        snprintf(field_path, sizeof(field_path), "data.%s._%d", vname, b);
        IronLIR_ValueId pval = iron_lir_get_field(ctx->current_func, ctx->current_block,
                                                    parent_val, field_path, payload_type, span)->id;

        IronHIR_Expr *nested = (pattern->pattern.nested_patterns &&
                                 b < pattern->pattern.binding_count)
                                ? pattern->pattern.nested_patterns[b] : NULL;

        if (nested && nested->kind == IRON_HIR_EXPR_PATTERN) {
            /* Recurse */
            emit_nested_pattern_bindings(ctx, pval, payload_type, nested, span);
        } else if (bname) {
            /* Simple binding */
            IronLIR_ValueId binding_alloca = emit_alloca_in_entry(ctx, payload_type, bname, span);
            iron_lir_store(ctx->current_func, ctx->current_block,
                           binding_alloca, pval, span);
            IronHIR_VarId bind_var = iron_hir_alloc_var(ctx->hir, bname, payload_type, false);
            hmput(ctx->var_alloca_map, bind_var, binding_alloca);
        }
        /* else: wildcard _ — skip */
    }
}
```

**Step 5: Handle binding variable references in arm bodies.**

The arm body references binding variables via `IRON_HIR_EXPR_IDENT`. The existing `lower_expr` for IDENT looks up `ctx->var_alloca_map` by VarId. However, the pattern bindings created in Step 2f use `iron_hir_alloc_var` to create new VarIds, and the arm body's IDENT nodes have the VarIds assigned during HIR lowering in hir_lower.c.

**Critical detail:** The binding variables in the arm body are referenced by the same names but may have different VarIds than the ones created in Step 2f. To fix this:
- In the IRON_NODE_PATTERN lowering in hir_lower.c (from Plan 01), the pattern bindings should inject `IRON_HIR_STMT_LET` nodes into the arm body block with VarIds that the body's IDENTs already reference.
- OR, in hir_to_lir.c, look up binding names in the arm body's IDENT nodes and match them to the alloca VarIds created during pattern extraction.

**The recommended approach** (from RESEARCH.md Open Question 1): In hir_lower.c, for each non-wildcard binding in a PATTERN, inject an `IRON_HIR_STMT_LET` at the START of the arm body block. The LET's init expression is a FIELD_ACCESS on the scrutinee. This way:
- The VarId is allocated during HIR lowering
- The arm body IDENTs reference this VarId
- In hir_to_lir.c, the existing STMT_LET handler calls `emit_alloca_in_entry` + store

This means Step 2f's manual binding handling may NOT be needed if the LET injection in hir_lower.c is done correctly. **Check Plan 01's implementation:** if hir_lower.c already injects LET nodes for pattern bindings, the arm body lowering in hir_to_lir.c will handle them automatically via the existing LET path. If not, add the LET injection in hir_lower.c as part of this task.
