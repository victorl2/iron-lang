# Iron v3.0.0-alpha — Method Ergonomics

*Released: 2026-04-23*

Iron v3.0 is the largest grammar overhaul since v1.0. Ten phases and 87
requirements rework how objects, methods, visibility, mutation, and
initialization are expressed. The core idea is simple: methods belong
inside the object they operate on, not scattered across the file as
free-standing receiver functions. Everything else follows from that.

The milestone spans phases 82 through 91, delivers the first fully
automated migration codemod (`ironc migrate`), and ships verified
byte-identical code generation relative to the v2.2 baseline via the
`verify-v3-migration.sh` release-gate script.

## BREAKING: v3.0 is not a superset of v2.2

v3.0 is the first Iron release where the pure-superset guard does not
hold. Three v2.2 constructs are parse errors under v3.0:

1. **Receiver-method syntax removed.** `func (p: Player) take_damage(n: Int)`
   and `func (mut p: Player) take_damage(n: Int)` are no longer valid.
   Methods now live inside `object T { ... }` blocks with implicit `self`.

2. **Inline field defaults removed.** `var health: Int = 0` at the
   field-declaration site is a parse error. Fields must be assigned
   in `init`.

3. **`mut` keyword removed.** Mutation is now the default at method
   declaration sites. Non-mutating methods opt in with `readonly` or
   `pure`.

**What to do before upgrading:**

```bash
# Pin to v2.2.0-alpha until you are ready to migrate
ironc --version   # confirm: 2.2.0-alpha

# When ready, run the codemod once against your sources
ironc migrate --from v2 --to v3 src/
```

The codemod is deterministic and idempotent. Re-running it on already-
migrated code is safe. A full step-by-step migration guide is at
`docs/site/migration-v2-to-v3.md`.

Do not upgrade to v3.0 silently. Pin, run the codemod, verify your
build, then cut over.

## Methods inside objects

Before v3.0, methods were written as free-standing receiver functions
anywhere in the file:

```iron
-- v2.2: receiver-method syntax (removed in v3.0)
object Player {
    var health: Int
    var name: String
}

func (p: Player) take_damage(n: Int) {
    p.health = p.health - n
}

func (mut p: Player) heal(n: Int) {
    p.health = p.health + n
}
```

In v3.0, methods move inside the `object` block. The receiver is always
named `self` and is implicit:

```iron
-- v3.0: methods declared inside the object block
object Player {
    var health: Int
    var name: String

    init(health: Int, name: String) {
        self.health = health
        self.name = name
    }

    func take_damage(n: Int) {
        self.health = self.health - n
    }

    readonly func is_alive() -> Bool {
        return self.health > 0
    }
}
```

Accessing a field inside a method requires the explicit `self.` prefix.
Writing `health` without `self.` is a compile error. Dispatch is
unchanged at call sites: `player.take_damage(5)` works exactly as before.

## Visibility -- single `pub` keyword

Everything in Iron is private by default in v3.0: fields and methods
alike. Opting into exposure uses a single keyword, `pub`.

```iron
object Account {
    var balance: Int       -- private: not visible outside Account
    pub var owner: String  -- public field with property syntax

    init(owner: String, balance: Int) {
        self.owner = owner
        self.balance = balance
    }

    pub readonly func balance_cents() -> Int {
        return self.balance * 100
    }

    -- private helper: only callable from inside Account
    func apply_fee(amount: Int) {
        self.balance = self.balance - amount
    }
}

func main() {
    var acc = Account("Alice", 500)
    println(acc.owner)               -- OK: pub field, property syntax
    println("{acc.balance_cents()}")  -- OK: pub method
    -- acc.apply_fee(10)             -- error: apply_fee is private
    -- acc.balance                   -- error: balance is private
}
```

`pub val field` synthesizes a read-only getter; `pub var field`
synthesizes both a getter and a setter. Call sites use property syntax
(`acc.owner`) rather than explicit getter calls.

## Mutation tiers -- default / readonly / pure

v3.0 replaces the v2.2 `mut`-receiver opt-in with a three-tier
declaration system. The default is mutating; non-mutating methods opt in:

| Tier | Declaration | Can write `self.field`? | Can do I/O or globals? |
|------|-------------|------------------------|------------------------|
| Default | `func name()` | Yes | Yes |
| Read-only | `readonly func name()` | No | Yes |
| Pure | `pure func name()` | No | No |

```iron
object Counter {
    var value: Int

    init(start: Int) {
        self.value = start
    }

    func increment() {
        self.value = self.value + 1    -- OK: default tier, mutating
    }

    readonly func current() -> Int {
        return self.value              -- OK: read-only, no writes
    }

    pure func doubled() -> Int {
        return self.value * 2          -- OK: pure, no side effects
    }
}
```

The compiler enforces tiers transitively:

```iron
-- error[E03F1]: readonly func cannot write to self.value
readonly func reset() {
    self.value = 0    -- rejected
}

-- error[E03F2]: readonly func cannot call mutating method
readonly func reset_via_call() {
    self.increment()  -- rejected: increment() is default (mutating)
}
```

A `readonly` method may call other `readonly` or `pure` methods on
`self`. A `pure` method may only call other `pure` methods. The full
enforcement graph is checked at compile time with source carets.

## Mandatory init

Every object in v3.0 must declare at least one `init`. Fieldless objects
get a synthesized empty init automatically.

Two init forms exist: anonymous (constructed via `Type(args)`) and
named (constructed via `Type.name(args)`):

```iron
object Vec2 {
    val x: Float
    val y: Float

    -- Anonymous init: constructed via Vec2(x, y)
    init(x: Float, y: Float) {
        self.x = x
        self.y = y
    }

    -- Named init: constructed via Vec2.zero()
    init zero() {
        self.x = 0.0
        self.y = 0.0
    }

    -- Named init with conversion: constructed via Vec2.from_int(n)
    init from_int(n: Int) {
        self.x = Float(n)
        self.y = Float(n)
    }
}

func main() {
    val a = Vec2(3.0, 4.0)     -- anonymous
    val b = Vec2.zero()        -- named
    val c = Vec2.from_int(5)   -- named with conversion
}
```

The compiler enforces definite assignment: every field must be assigned
on every exit path through the init body, or it is a compile error
(E03XZ). `val` fields may only be assigned once (E03YY). Calling a
method on `self` before all fields are assigned is rejected (E03YX).

Fieldless objects need no init declaration:

```iron
object Marker {}    -- synthesized empty init; Marker() is valid
```

## Open extension via patch

`patch object T { ... }` adds methods and inits to any type, including
types you do not own and built-in primitives. Patches are program-wide.

```iron
-- Extend the built-in Int type with a custom method
patch object Int {
    pub readonly func double() -> Int {
        return self * 2
    }

    pub readonly func clamp(lo: Int, hi: Int) -> Int {
        if self < lo { return lo }
        if self > hi { return hi }
        return self
    }
}

func main() {
    println("{5.double()}")          -- 10
    println("{(-3).clamp(0, 10)}")   -- 0
    println("{200.clamp(0, 100)}")   -- 100
}
```

Patches may also add named inits and declare retroactive interface
conformance:

```iron
interface Describable {
    readonly func describe() -> String
}

patch object Int implements Describable {
    pub readonly func describe() -> String {
        return "Int(" + self.to_string() + ")"
    }
}

func print_desc(d: Describable) {
    println(d.describe())
}

func main() {
    print_desc(42)    -- Int(42)
}
```

Rules: patches may not add fields or change the visibility of existing
members. Duplicate method signatures across patches for the same type
are a compile error (E03XX). There is no priority or override mechanism;
collisions must be resolved by the author.

## Interfaces and Self

Interfaces in v3.0 support mutation-tier modifiers on method signatures
and default implementations:

```iron
interface Shape {
    readonly func area() -> Float
    readonly func perimeter() -> Float

    -- Default implementation: types may override, but need not
    readonly func describe() -> String {
        return "shape with area " + self.area().to_string()
    }
}

object Circle {
    val radius: Float

    init(radius: Float) {
        self.radius = radius
    }

    readonly func area() -> Float {
        return 3.14159 * self.radius * self.radius
    }

    readonly func perimeter() -> Float {
        return 2.0 * 3.14159 * self.radius
    }

    -- Overrides the default implementation
    readonly func describe() -> String {
        return "circle r=" + self.radius.to_string()
    }
}
```

The `Self` return type refers to the concrete type of the receiver,
enabling type-safe builder chains and copy constructors:

```iron
interface Clonable {
    readonly func clone() -> Self
}

object Point {
    val x: Float
    val y: Float

    init(x: Float, y: Float) {
        self.x = x
        self.y = y
    }

    readonly func clone() -> Self {
        return Self(self.x, self.y)    -- Self(args) uses the anonymous init
    }

    readonly func with_x(x: Float) -> Self {
        return Self(x, self.y)
    }
}

func main() {
    val p = Point(1.0, 2.0)
    val q = p.clone()
    val r = p.with_x(5.0)
}
```

`Self.name(args)` uses a named init. `Self` is resolved at the call
site to the concrete type of the receiver, so interface-declared
`Self`-returning methods work correctly through dynamic dispatch.

## Migration

Run the codemod once against your Iron sources:

```bash
ironc migrate --from v2 --to v3 src/
```

The codemod covers three transforms automatically:

1. **Receiver methods to in-block.** `func (p: Player) method()` moves
   inside the `object Player` block and the receiver is renamed `self`.
2. **Mut receivers to default tier.** `func (mut p: Player) method()`
   becomes a plain `func method()` inside the block (mutation is the
   default in v3.0).
3. **Inline field defaults to init.** `var health: Int = 0` at field
   declaration sites is extracted to an `init` body.

The codemod is idempotent: re-running on already-migrated sources
produces no changes. It emits a unified diff on stderr for review before
applying. Generated C output for migrated code is byte-identical to the
v2.2 baseline, verified by `scripts/verify-v3-migration.sh`.

For a full step-by-step guide covering install, backup, codemod, manual
TODO resolution, and troubleshooting, see
[docs/site/migration-v2-to-v3.md](../site/migration-v2-to-v3.md).

**Pin guidance:** Keep `v2.2.0-alpha` pinned until you have run the
codemod and verified your build passes. There is no silent upgrade path.
